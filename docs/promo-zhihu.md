# 我写了 500 章网文之后，把方法论做成了一个开源项目

## 先给结论

我花了两年时间，断断续续写了 500 章网文。其中一本 500 章的全书被我自己判了死刑——不是因为故事不好，是因为 AI 味太重、节奏不对、中段疲软，改不回来了。

那本书死了之后，我把踩过的每一个坑都记录下来，一步步迭代出了一个写作体系。现在我把这套东西开源了。

**GitHub 地址：https://github.com/LAY-lgtm/novel-writing-framework**

---

## 这个项目是什么？

一套**工程化的网文写作方法论**。如果你写过网文，你应该经历过这些：

- 写了 100 章回头看，前 50 章想重写
- 读者说 "AI 味很重"，但你自己看不出来
- 中段（100-300 章）不知道该怎么推进
- 发书之后数据不好，但不知道为什么

这个项目试图解决这些问题。它不是 "怎么写好小说" 的文学理论——它是 "怎么让一本网文不低于某个质量底线" 的工程框架。

---

## 几个比较有意思的点

### 1. "不是 X 而是 Y" —— 全书最大的 AI 指纹

我的一本小说初稿里出现了 663 次 "不是……而是……" 句式。663 次。我花了数十轮交互才清洗到 31 次。

为什么这么多？因为 AI 默认就会写这个句式——它是 AI 做判断对比时最自然的表达方式。但读者读 600 多次 "不是 A 是 B"，潜意识会觉得 "这不像人写的"。

现在我每写 3 章就扫描一次这个句式，超过 2 次 / 章立刻修。从源头控制比后期清洗省 10 倍时间。

### 2. 番茄和起点根本不是一种写法

大多数网文教程不管平台，给的建议是通用的。但番茄和起点的读者行为完全不同：

- **番茄**：3 秒划走。爽点密度要 3 章一爽。情感只吃撕裂式爆发。变现靠广告分成 + 全勤。
- **起点**：愿意养书。允许慢热铺垫。群像可以展开。读者会考据你的世界观设定。

如果你用起点的慢节奏去冲番茄的 3 秒法则，开篇前 300 字没炸就直接凉。如果你用番茄的全程碾压去冲起点的群像读者，他们会觉得配角全是工具人。

这个项目给两个平台分别写了完备的技能文件。

### 3. 三章一轮 —— 从血的教训里长出来的制度

以前我每 10 章做一次自检。663 次 "不是 X 是 Y" 就是这么积累出来的——如果能每 3 章扫一次，第一轮就会发现异常。

现在我的流程是：每写 3 章 → 停笔 → 技术自检（CJK 字数 / 破折号 / 动词开头 / 钩子）→ 平台质量自检 → AI 指纹扫描 → 不合格当场修。

**不要攒到 50 章一起修。后期修复的成本是指数级的。**

### 4. 每条规则背后都有一个已修复的错误

- "连续 17 章在同一场景" → 加了场景多样性检查，连续 3 章同一场景必须切换
- "连续三章结尾重复同一个长线悬念" → 加了长线钩子间隔 ≥ 2 章的硬指标
- "正文里出现了'卷一''第X章'等元话语" → 加了元话语禁令
- "感叹号和粗口全是后期注入的，效率极低还串音" → 加了初稿到位原则

这不是一个 "我觉得应该这样写" 的项目。这是一个 "我犯过这个错，修起来很痛，所以你不用再犯" 的项目。

## 项目结构

```
├── writing-novel.md      基础层：14 阶段通用流程
├── piqie-writing.md      番茄平台专项 + 变现策略
├── qidian-writing.md     起点平台专项 + 付费设计
├── zhi-dou-writing.md    智斗类型专项（四大名家技法融合）
└── novel-improver.md     双平台改良诊断（13 项量化指标）
```

总计约 193KB 文檔，MIT 开源。

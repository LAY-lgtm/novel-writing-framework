# 分享一个自己做的网文写作开源框架（500 章实战提炼）

## 先说背景

写了两年网文，五百多章。一本废了，一本在写中。

废掉的那本不是因为故事不好——纯粹是被 AI 味和中段疲软拖死的。回头看的时候发现光 "不是 X 而是 Y" 这个句式就出现了 663 次。读者说 "像 AI 写的"，我当时还不服——现在回头看，人家说得对。

那本书死了之后，我把每次审稿发现的问题、每次改文总结的规则、每个平台的差异认知，全部沉淀成了一套方法论。不只是 "写得更好" 的文学建议——是能执行的、有硬指标的、可重复使用的写作流程。

**GitHub 地址：https://github.com/LAY-lgtm/novel-writing-framework**

## 这套东西到底说了什么

### 1. 番茄和起点是两种完全不同的写作

这不是 "番茄节奏更快" 这种废话。是具体到：

| 维度 | 番茄 | 起点 |
|------|------|------|
| 读者判断一本书的时间 | 3 秒 | 前 20-50 章 |
| 爽点密度 | 3 章一爽 | 可以 10 章铺垫 |
| 群像 | 主角视角为主 | 配角必须有独立高光 |
| 读者考据能力 | 不会考据 | 会截图存档挑逻辑漏洞 |
| 情感模式 | 撕裂式爆发 | 安静感动也可以 |
| 变现方式 | 广告分成 + 全勤 | 章节订阅 |

**如果你用写起点的方式去写番茄——开篇前 300 字没有物理事件，开头三章在铺垫世界观——读者 3 秒划走。验证期 3 天没数据直接断流。**

反过来，如果你用写番茄的方式去写起点——全程主角碾压、配角全是工具人、世界观过于简化——起点读者会在养肥期默默弃书，你永远不知道他们什么时候走的。

### 2. "不是 X 而是 Y" 是 AI 味的头号标志

这个句式我统计了：AI 生成的文章里，"不是 X 而是 Y" 的自然出现频率远高于人类写作。

我的做法：
- 每写完 3 章 grep 扫描一次
- 超过 2 次 / 章立刻改
- 对话中的 "不是" 保留（口语自然），叙事中的砍

从 663 次洗到 31 次的代价太大了——初稿控制住的成本是后期清洗的十分之一。

### 3. 配角工具人的检测方法

起点读者追群像文的核心心理是 "这群人在一起才无敌"。如果只有主角一个人在发光，配角就变成了跟班。

我的检测方法：
- 每 10 章查一次：每个配角有没有独立的、不依附主角的高光瞬间？
- 有没有配角连续 10 章以上只是 "站在那里"？
- 每个主要配角在每 20 章里有没有至少一个 "和你无关但我想知道" 的独立故事碎片？

如果删掉一个配角这 10 章没有任何不同——这个配角没有存在的必要。

### 4. 三章一轮制度

以前我 10 章一查。问题发现太晚。

现在：写 3 章 → 停笔 → 技术自检（CJK/破折号 / 动词 / 钩子）→ 平台质量自检 → AI 指纹扫描 → 不达标当场修。

663 次 "不是 X 是 Y" 如果每 3 章扫一次，早就发现了。

## 项目包含什么

```
├── writing-novel.md      ← 基础层：14 阶段通用流程（500+ 章凝练）
├── piqie-writing.md      ← 番茄专项 + 变现策略 + 算法适配
├── qidian-writing.md     ← 起点专项 + 付费设计 + 群像规则
├── zhi-dou-writing.md    ← 智斗类型专项
└── novel-improver.md     ← 双平台改良诊断
```

193KB 文檔，MIT 开源。

目前主要作者的实战经验集中在东方玄幻品类，其他品类（都市、历史、悬疑）的实战数据还比较少。如果论坛里有写其他品类的朋友，欢迎贡献。

> **一套从 500+ 章实战中提炼的中文网文写作全流程框架。不只是"怎么写"——从平台选型、逐章写作、质量自检、平台变现，到数据驱动的持续改良。**

## 这套方法论能解决什么问题？

写过网文的人都知道：
- 写了 100 章回头看，发现前 50 章都是垃圾
- 剧情写到中段突然不知道接下来写什么
- 读者说 "AI 味很重"，但你自己看不出来
- 发了书但不知道算法怎么推，为什么数据不好

这套方法论从根源上解决这些问题。

**适用范围**：20 万字到 500 万字的各类长篇网文，支持番茄小说、起点中文网双平台。

## 核心方法

### 1. 写作前：平台 × 类型决策

| 维度 | 番茄爆款 | 起点爆款 |
|------|---------|---------|
| 读者耐心 | 3 秒划走 | 愿意读慢热设定 |
| 爽点密度 | 3 章一爽 | 可以 10 章铺垫 |
| 主角 | 全程碾压 | 允许恐惧、犯错、成长 |
| 群像 | 主角视角为主 | 群像充分展开 |
| 情感 | 撕裂式爆发为主 | 安静感动也可以成立 |
| 变现 | 广告分成+全勤 | 章节订阅 |

### 2. 写作中：三章一轮

每写 3 章立即停笔自检——不要攒到 50 章一起修。

### 3. 写作后：双平台改良诊断

10 维评分体系 + 13 项硬性量化指标 + 读者模拟评审。

## 从实践中学到的核心教训

### "不是 X 而是 Y"——全书最大的 AI 指纹

实战项目中出现了 **663 次**，清洗到 **31 次**，耗费数十轮交互。是所有修复中工作量最大的单项。

### 663 → 31 是如何发生的？

因为每 10 章才自检一次。如果每 3 章扫描一次，第一轮就会暴露。

**这催生了整个技能体系最核心的制度：三章一轮。**

### 其他血的教训

- 正文中绝对不能出现 "卷一""第 X 章""前文""后文" 等元话语——读者瞬间出戏
- 连续 17 章在同一场景 → 读者闭眼就是同一画面 → 必须维护 ≥5 个场景轮换
- 长线钩子连续 3 章结尾重复 → 读者麻木，变成空炮
- 感叹号和粗口必须初稿就放——后期注入效率极低且容易角色串音
- 每个作者都有"默认倾向"和平台要求是 180 度反向的——写前阻断比写完检查有效

## 快速开始

### 1. 确定你要什么

```bash
# 我要写番茄快节奏爆款
→ 加载 writing-novel + piqie-writing

# 我要写起点慢热品质文
→ 加载 writing-novel + qidian-writing

# 我要写智斗权谋
→ 在上面的平台选择基础上 + zhi-dou-writing

# 我要改良一本已写完的书
→ 加载 novel-improver（独立运行）
```

### 2. 创建项目配置

参考 `skills/qidian-writing.md` 第十一部分的 CLAUDE.md 项目配置模板，复制到你的小说项目根目录。

### 3. 开始写作

按照技能文件中的流程执行：

1. **动笔前**：单章阻断五问（不通过不准写）
2. **三章一轮**：写完 3 章 → 技术自检 → 平台质量自检 → AI 指纹扫描
3. **每 10 章**：批次审查 → 爽点分布 → 配角检查 → 修改
4. **每卷**：全卷一致性审查

## FAQ

**Q: 我是纯新人，该从哪个技能开始？**
A: 先读 `writing-novel.md` 的前两部分（核心教训 + 风格参考）和 `piqie-writing.md`（如果你选番茄）或者 `qidian-writing.md`（如果你选起点）。然后按"快速开始"创建项目配置。

**Q: 这些规则会不会限制创意？**
A: 规则是地板不是天花板。它们保证你的书不低于某个质量底线——在底线之上，创意是自由的。

**Q: 我需要会用 Claude Code 才能用这套方法论吗？**
A: 不一定。所有技能文件都是纯 Markdown，可以手动执行也可以让任何 AI 助手加载执行。

**Q: 这套方法论写出来的书有没有"套路感"？**
A: 如果你的书有套路感，不是规则太多，是你的角色不够真。这套方法论里有一个完整的"真人感"子系统（角色语音卡、独立故事碎片、低谷时刻），专门解决这个问题。

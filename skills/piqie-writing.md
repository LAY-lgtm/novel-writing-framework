# 番茄爆款写作技能（融合版）

当用户请求创作番茄小说新章节时，必须按以下流程执行。本skill融合了500章+350章两次长篇实战的全部教训。

---

## 核心元教训

**我的默认写作倾向和番茄爆款要求是180度反向的。**

```
我的默认模式：静谧→对话→发现→理解→和解→抒情收束
番茄爆款要求：冲突→战斗→反杀→碾压→生死局→炸弹收束
```

这不是"需要调整"的偏差，是两极对立。每次写新章节时默认倾向会自动覆盖所有规则。**唯一有效的对抗手段：写每一章之前强制执行阻断五问。不通过不准动笔。**

---

## 一、写前单章阻断五问（每章动笔前必须过）

```
问1：这一章反派造成了什么可见伤害？
      必须具体到：谁受伤了、什么东西被破坏了。
      答不出→不准写。

问2：这一章的物理对抗是什么？
      手碰到金属/火焰/焊条/枪杆/拳头——纯对话不算。
      答不出→不准写。

问3：这一章的爽点能不能用"主语+动词+宾语+对方损失"描述？
      不能用"发现了XX""理解了XX""对话了XX"——这些不是爽点。
      答不出→不准写。

问4：这一章结尾最后一句话会不会让读者心跳加速？
      如果读完感觉"安静""温暖""感动"→不合格。
      合格的感觉是"他要干什么？""下一页！""完了！"。
      不合格→不准写。

问5：这一章前300字有没有物理事件？
      如果前300字是"XX站在XX看着XX想着XX"→读者3秒划走。
      必须有：炸了/断了/打了/冲了/倒了/烧了。
      没有→不准写。
```

## 二、写前10章强制自检（输出大纲时必须呈现）

在写任何一个字之前，必须输出大纲表格：

| 章 | 标题 | 分类(战斗/紧张/生活) | 爽点类型 | 战斗场景 | 主角参与方式 | 结尾钩子类型 |
|---|------|------|------|------|------|------|
| N | XXXX | XXX | XXX | XXX | XXX | XXX |

### 大纲强制检查项（全部通过才能开始写作）：
- [ ] **战斗章≥3？紧张章≥3？生活章≤2？**
- [ ] **六种爽点全部轮换？**（击杀→反转→牺牲→觉醒→配合→智斗，禁止连续同一类型）
- [ ] **战斗场景不连续重复？**（禁止连续两场同一场景）
- [ ] **主角在至少1章中有主动决策或亲手战斗？**（不是放壳/看数据/说一句话）
- [ ] **这一批中至少1章有人受伤或流血？**
- [ ] **如果全书已有80+章无牺牲——这一批必须安排牺牲或埋牺牲伏笔**
- [ ] **1-2章内必须有正面爆发？**（不能把爆发全堆在后面）
- [ ] **每10章至少1次碾压或装逼？**（主角或核心角色展示压倒性实力）
- [ ] **章末钩子选项轮换？**（危机/反转/悬念/战斗钩子——不用"明天继续"）
- [ ] **反派在前1-2章内已主动出手造成可见伤害？**

## 三、通用写作质量铁律（所有平台通用，来自《黑龙醒》663次→31次的清洗教训）

### 铁律1：禁止AI句式——"不是X，是Y"是全书最大的AI指纹

- **事实**：《黑龙醒》从663次"不是X是Y"清洗到31次，耗费数十轮交互——是所有修复中耗时最长的单项
- **写作时自觉避免**：不要用"不是A，是B"做判断对比。直接写B。如果必须对比，用"——"或另起一句
- **每写完一批立即grep扫描**，超过2次/章立刻回头改
- 对话中的"不是"可以保留（口语自然），叙事中的必须砍

### 铁律2：正文中绝对禁止出现元话语

- **绝对禁止在正文中出现**："卷一""卷二""第X章""前文所述""后文再表""本章"
- 这些是作者/编辑视角的文字，读者看到会立刻出戏
- 引用过去事件时，用具体内容描述（"城寨那夜""箭穿肩之后"），不用章节编号

### 铁律3：感叹号和粗口必须在初稿中就到位

- 后期注入效率极低且容易出bug（角色串音）
- 战斗章每章≥2个感叹号（初稿就写出来）
- 每个会爆粗的角色按角色语音卡来——**初稿就要有，不要等后期补**

### 铁律4：场景不能钉死在同一个地方

- 连续3章以上在同一核心场景 → 必须切换
- 维护场景轮换池≥5个可用场景
- 每次场景切换顺带插入其他视角/线，保持多线叙事

### 铁律5：长线钩子不能每章结尾都提

- 长线倒计时/悬念每隔3-4章提一次就够
- 中间用其他类型的钩子——变成空炮读者就麻木了
- 检查：列出最近5章的结尾钩子，长线钩子间隔必须≥2章

### 铁律6：破折号控制必须自动化

- 每次重写/扩充都会导致破折号超标（严重时一章102个破折号）
- **不再手工处理破折号**——脚本批量替换为中文逗号
- 对话中的破折号（打断/转折）优先保留，叙事中的优先替换

### 铁律7：去AI化要在初稿就做对——三章一轮扫描

- 后期修复AI味的成本是初稿的10倍以上
- **每写完3章立即跑AI指纹扫描**：不是X是Y次数、感叹号数量、粗口数量、AI口水词密度（缓缓/不由得/眼底闪过/心中升起/说不出的/这意味着）
- 不达标的当场修，**不要攒到51章一起修**

### 铁律8：创建角色语音卡

- 为主要角色（主角+2-3个核心配角）创建语音卡，定义：口头禅/高频词、粗口风格、说话节奏
- 每次写角色对话时对照语音卡——确保每个人说话方式不一样
- 避免所有角色用同一种语气说话（这是AI味的重灾区）

## 四、番茄小说特有的五大禁令（违反=重写）

这些是从350章审稿中发现的专属病症，普通写作规则覆盖不到：

### 禁令1：禁止无正面角色死亡
- 每个大篇章（约80章）必须有至少1次永久性牺牲
- 如果全书已有80+章无牺牲——下一批必须安排牺牲或至少埋牺牲伏笔

### 禁令2：禁止主角变成旁观者
- 主角必须在每10章中至少1次：亲手战斗 / 主动做决策 / 独立行动线
- 禁止行为：放技能在X上、看数据、说一句话、转某样物件
- 他的成长线：第一次手指发抖→第二次不抖了→后来果断

### 禁令3：禁止X0收束章使用群像模板
- 禁止：所有人都轮一遍的群像模板（角色A归档→角色B放武器→女孩画画→保安调频→清洁工合手册→主角转物件→明天继续）
- 每章必须有不同的收束方式，聚焦1-2个角色
- 至少一半X0章中避免重复使用同一个标志性意象

### 禁令4：禁止"安静的温馨"成为唯一情感底色
- 每10章必须有至少1次撕裂式情绪爆发（愤怒/崩溃/大哭）
- 不能全是：蒸包子/补衣服/画画/擦武器等日常琐碎

### 禁令5：禁止所有反派都不会说话
- 必须至少有一个能对话博弈的人类/人形反派
- 不能全是不会说话的怪物

## 五、番茄爆款通用规则（源自500章实战）

### 节奏铁律
```
3章一小爽：每3章一次情绪释放（打脸/碾压/反转/救人）
5章一大爽：每5章一次高潮（生死局/反杀/权力登顶/系统崩退）
10章一高潮：每10章一次单元级高潮
余韵≤2章：高潮后日常最多2章，第3章必须出现新冲突
```

### 爽点直给法则
番茄读者要的七种爽：打脸、碾压、装逼、截胡、复仇、升级、反转
禁止文青爽：理解、和解、发现、放下、开门、联网、对话

**爽点验证标准**：能用"谁+动词+谁+损失"描述。损失=端口崩了/攻击灭了/武器废了/人被击退/甲壳碎了/血溅了。

### 结尾钩子铁律
```
结尾只写四种：危机突降、悬念反转、挑衅叫板、死亡倒计时
禁止：抒情、安静、画面定格、感怀、明天继续、继续
单元终章结尾必须同时：收束本单元 + 炸开下一单元（新威胁必须有具体倒计时）
```

### 反派压迫感
- 反派必须在第1-2章内造成可见伤害
- 每10章至少1次"读者真的觉得主角会输/会死"的生死局
- 可见伤害=能用手机拍照拍到的：骨头断了/货柜砸了/钢缆崩了/墙面熔了

### 开章炸裂
- 开章公式：敌人用X攻击了Y，造成了Z的伤害，角色反击
- 禁止开章公式：角色X发现了Y，这个Y意味着Z，他们决定...

### 智斗的正确用法
- 智斗是前菜不是主菜，不超过一章
- 智斗后立即接正面爆发
- 一章想、一章做、一章炸

## 六、写作流程（10章一批 + 每3章扫描）

```
1. 输出10章大纲表格（标注所有维度）
2. 大纲自检（对照"大纲强制检查项"——全部通过才能动笔）
3. 逐章创作（每章动笔前过"单章阻断五问"）
4. 每写完3章立即跑AI指纹扫描（不是X是Y、感叹号、粗口、AI口水词——不达标当场修）
5. 第一轮技术自检（CJK≥2000、Dash≤20、开头80CJK动词、每章结尾是钩子）
6. 修复破折号（脚本批量替换→中文逗号，不手工处理）
7. 扩充CJK不足章
8. 第二轮质量自检（对照"穿书反派五大禁令"和"禁止清单"）
9. 第三轮全章扫描确认（全部通过CJK/Dash/Verb/钩子/场景多样性/长线钩子间隔）
10. 生成Word文档
11. 更新牺牲爽追踪记录
```

### 角色语音卡规则（每出现新角色即创建）
- 为主要角色定义：口头禅/高频词、粗口风格、说话节奏（短句/长句/句式偏好）
- 每次写角色对话时对照语音卡——确保每个人说话方式不一样

## 九、番茄平台变现与算法策略

### 核心变现模式

番茄采用 **"免费阅读 + 广告变现（IAA）"** 模式。读者免费看书，平台通过广告盈利，再与作者分成。**只有签约才有收益和流量扶持。**

### 收入构成

| 收入来源 | 占比/金额 | 说明 |
|----------|----------|------|
| **听读广告分成** | 约55%，核心收入 | 按有效阅读时长+完读率+广告曝光结算，独家签约通常拿50% |
| **全勤奖** | 600-800元/月 | 新人最稳收入。日更4000字=600元/月，日更6000字=800元/月。门槛：月更≥10万字 + 当月听读收益≥500元 |
| **版权衍生** | 增长最快 | 短剧改编保底+分成，头部单部50万-300万 |
| **平台激励** | 季度奖/半年奖/全年奖 | 2026年大幅升级，全年奖上限88万元 |
| **短篇保底** | 80-700元/千字 | 短篇分成比例10:0（作者拿100%） |

### 全勤奖——新人最重要的稳定收入

**这是大二学生最需要关注的一项。** 全勤奖是固定收入，只要你按时按量更新，这600-800块是跑不掉的。

| 档位 | 基础金额 | 浮动上限 | 门槛 |
|------|----------|----------|------|
| 日更4000字 | 600元/月 | +听读分成×5%（上限1500元） | 月更≥10万字，当月听读收益≥500元 |
| 日更6000字 | 800元/月 | +听读分成×5%（上限2000元） | 月更≥10万字，当月听读收益≥500元 |

**策略**：新人先冲日更4000字档，稳拿600元/月。日更6000字档的800元需要投入更多时间，性价比对新手来说不如用余力提升内容质量。600元/月 × 17个月 = 10200元——靠全勤奖就能突破1w。

### 番茄算法机制——怎么让算法推你

#### 核心数据指标（按重要性排序）

| 指标 | 权重 | 说明 |
|------|------|------|
| **首章完读率** | ⭐⭐⭐⭐⭐ | 读者看完第一章后是否翻到第二章——最重要的生死线 |
| **前3章完读率** | ⭐⭐⭐⭐⭐ | 脑洞文需15%以上，传统文需10%以上；低于35%追读→立刻优化开头 |
| **章节连续点击率** | ⭐⭐⭐⭐ | 看完一章是否立刻点下一章——比收藏重要10倍 |
| **追读率** | ⭐⭐⭐⭐ | 低于20%无法进入大流量池；优质网文追更衰减点在第7/21/45章 |
| **有效阅读时长** | ⭐⭐⭐ | 系统过滤后台/挂机，只计算真正在阅读的时间 |
| **互动数据** | ⭐⭐⭐ | 加入书架、催更、评论、段评的数量和质量 |
| **次日留存** | ⭐⭐⭐ | 读者第二天是否回来看 |

#### 流量两阶段

**阶段一：验证期（约3-7天）**
- 字数达2万字可申请签约，8万字进入验证期（2025年新规）
- 初始分500-5000人次测试，考核书名/简介/封面/开头的吸量能力
- **关键变化**：验证期实际只有3天左右就定生死，数据不达标直接断流
- 吸量等级：D级(0-100阅读) → SSS级(2000+阅读)，首日吸量400以上为S级

**阶段二：首秀期（14天）**
- 触发条件：10万字 + 连续更新
- 每日与同类作品PK，PK指标：吸量/追更率/留存率
- 首秀数据优秀→持续14天推荐；数据差→可能仅1天就断流
- 首秀结束后进入"静默期"，根据长期数据决定后续流量

#### 算法适配写作策略

| 策略 | 做法 | 对应指标 |
|------|------|---------|
| **第一章炸裂** | 前300字必须有物理事件，3秒抓住注意力 | 首章完读率 |
| **前三章密度拉满** | 每章结尾都是强钩子，让读者不得不点下一章 | 前3章完读率 + 章节连续点击率 |
| **第7/21/45章埋爆点** | 这些是番茄用户追更衰减的自然节点，必须有不能跳过的内容 | 追读率 |
| **章章结尾炸** | 一档结尾≥80%——每章最后一段让读者心跳加速 | 章节连续点击率 |
| **引导互动** | 章末加"觉得XXX的扣1"、设悬念让读者在段评讨论 | 互动数据 |
| **稳定更新** | 首秀前存稿5万字以上，推荐期日更6000字（分3章） | 追读率 + 全勤奖 |

#### 书名/简介/封面——被低估的流量入口

番茄的推荐流中，读者第一眼看到的是：**书名 + 封面 + 简介首句**。这三样比你的正文更早接触读者。

**书名策略**：
- 带搜索热词——番茄用户大量通过搜索找书
- 例：`末世：开局觉醒SSS级异能` 比 `末日之后` 好100倍
- 让人看一眼就知道这本书讲什么

**简介策略（番茄版）**：
- 第一句必须展示冲突或悬念——不能是世界观介绍
- 3-5句交代：主角困境 + 金手指 + 第一阶段目标
- 结尾暗示更大世界/更强敌人

**封面策略**：
- 不要自己设计——淘宝/闲鱼找专业做网文封面的，一幅30-80元
- 封面要一眼看得出品类（玄幻/都市/末世）
- 标题字要大、要清晰——手机上的缩略图就是指甲盖大小

#### 听书模式适配

番茄有大量用户是**听书**而不是看书的（通勤/睡前/做家务）。这意味着：

- **对话要像真的在说话**：读出来不别扭。写完对话后自己念一遍
- **段落不能太长**：听书模式下，一段超过100字听到后面已经忘了前面的主语
- **避免大段纯设定描述**：听书用户无法"扫一眼跳过"——他们必须听完
- **战斗场景的音效感**：用拟声词和节奏短句，听着有画面冲击力
- **粗口要写到能念出来的程度**：番茄的AI配音会念粗口，但过于夸张的会被和谐

## 十、新人破零路径（从0到签约收入）

### 第一步：定位选材（1周）

- 选一个番茄热门的品类：末世/脑洞/系统/重生/规则怪谈
- 去番茄App刷对应品类榜单前20名的**书名和简介**，感受"什么标题能让人点进去"
- 用写作技能的大纲设计做出10章规划

### 第二步：攒稿（2-3周）

- 攒到8-10万字（约40-50章），确保首秀期有足够存稿
- 每写完3章跑AI指纹扫描，确保质量
- 前3章反复打磨——这是验证期的生死线

### 第三步：发布+签约（第3-4周）

- 在番茄作家助手App发布，每天发2-3章
- 达2万字后申请签约
- 签约后继续稳定更新，等待8万字验证期

### 第四步：验证期+首秀（第4-8周）

- 8万字进入验证期，关注首章完读率和追读率
- 数据不理想→立刻优化前3章，换书名/简介
- 10万字首秀期全力更新，日更6000字分3章

### 第五步：拿全勤+看长线（第2个月起）

- 达标→每月稳定600-800元全勤奖
- 关注完读率趋势，在第7/21/45章节点埋爆点挽留
- 持续写到100章+，数据稳定后考虑开新书或写短篇

### 现实主义时间表

| 时间 | 里程碑 | 预计月收入 |
|------|--------|-----------|
| 第1-2个月 | 签约+首秀 | 0-300元（听读分成不稳定） |
| 第3-4个月 | 首秀后稳定期 | 全勤600元 + 听读分成200-500元 |
| 第5-6个月 | 累积100章+ | 全勤+听读800-2000元 |
| 累计第6-17个月 | → 突破1万总收入 | 全勤每月600×17=10200 |

**靠全勤奖+基础听读分成，17个月可以稳定突破1万。** 这不是暴富路径，但它是确定性最高的路径。

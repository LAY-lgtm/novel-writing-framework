# 小说改良Skill（番茄+起点双平台）

## 核心目标

将任意长篇小说系统性改良至目标平台的高标准。**动笔前必须确认目标平台**——番茄和起点的改良方向不同。

| | 番茄目标 | 起点目标 |
|------|----------|----------|
| 评分标准 | 9.0+评分 | 精品频道/万订 |
| 核心指标 | 完读率>70% | 订阅留存>70% |
| 改良重心 | 爽点密度+开篇抓人 | 信息增量+人物深度 |
| 文风方向 | 更直给、更炸 | 更有质感、更自洽 |

这不是简单的文字润色——而是从结构、人物、节奏、情感、语言五个维度进行深度诊断和精准改写。

## 番茄平台9.0+评分机制

### 评分构成（权重估计）

| 因素 | 权重 | 9.0+门槛 |
|------|------|----------|
| 完读率（读完/点开） | ~35% | >70%，核心指标 |
| 追读率（持续阅读用户占比） | ~25% | 每日追读>50%用户 |
| 互动率（评论+点赞+礼物） | ~15% | 每章评论>50条 |
| 评分分布 | ~15% | 5星>80%，1-2星<5% |
| 留存率（次日/7日/30日） | ~10% | 次日>80%，7日>60% |

### 完读率是核心

番茄的推荐算法极度依赖完读率。一本书评分高，首先是因为"更多人读完了"。影响完读率的关键因素：

1. **前3章定生死**——用户点开后只有3章的机会让他留下来
2. **每章结尾必须有钩子**——不钩住读者，他就会划走
3. **中段不能疲软**——第100-300章是完读率杀手区
4. **高潮必须有回报**——读者忍耐铺垫是为了高潮的爽，高潮不爽=弃书
5. **结尾必须满意**——差评主要来自"烂尾"，结尾不满意会拉低整体评分

### 番茄读者画像

- 阅读场景：通勤、睡前、摸鱼、饭后
- 阅读习惯：快速滑动，注意力窗口约15-30秒/章
- 偏好：强情绪驱动、爽点密集、人物有记忆点、对话多于描写
- 雷点：AI味（模板化语言）、水字数、人物崩坏、逻辑断裂、烂尾
- 高付费意愿场景：人物高光时刻、反转、收线、情感爆发

## 十维评审体系

对每一章、每一单元、每一卷进行十维评分，满分10分：

### 1. 开篇吸引力（Hook Strength）
- 第一章是否在3段内建立了核心悬念？
- 每个新单元的开篇是否重新抓住了读者？
- 每章开头是否避免了"温水煮青蛙"式慢热？

### 2. 人物塑造深度（Character Depth）
- 每个主要人物是否有独立的动机系统？
- 人物行为是否有内在一致性（不是情节需要就随便变）？
- 人物的缺点和局限是否真实可感？
- 配角是否有独立人格，而非主角的工具人？

### 3. 情节推进节奏（Pacing）
- 信息密度是否均匀（不能一章全是信息，下一章全是水）？
- 是否有"废章"（删掉不影响理解）？
- 爽点分布是否合理（不能连续10章没有爽点）？

### 4. 情感共鸣强度（Emotional Resonance）
- 读者能否在关键场景产生"我也有过这种感觉"的共鸣？
- 情感线是否真实（不狗血、不强行煽情）？
- 是否有"金句"级别的台词或叙述？

### 5. 世界观完整性（World Coherence）
- 世界观规则是否自洽（不允许为了情节随便改规则）？
- 读者是否需要大量前置知识才能理解？
- 世界观的展示方式是否自然（不是教科书式灌输）？

### 6. 语言风格独特性（Style Uniqueness）
- 是否有独立的叙事声音（不是AI能模仿出的模板风格）？
- 对话是否有区分度（不同人物说话方式不同）？
- 描写是否有画面感（不是空洞的形容词堆砌）？

### 7. 爽点密度与分布（Payoff Density）
- 每章是否有至少一个"读者会记住的点"？
- 爽点类型是否多样化（不只是打脸和升级）？
- 爽点是否有递进（不是同一模式的重复）？

### 8. 悬念与期待感（Suspense & Anticipation）
- 每条伏笔是否有清晰的投放→暗示→回收路径？
- 章节结尾是否让人"想翻下一页"？
- 悬念是否分层（短期待（章级）+ 中期待（单元级）+ 长期待（卷级））？

### 9. 章节结尾钩子（Chapter-End Hook）
- 每章结尾是否有钩子（不一定是悬念，可以是情感停顿、画面定格、金句）？
- 钩子类型是否多变（不能每章都用"但他不知道的是……"）？

### 10. 主题深度与格局（Theme & Vision）
- 小说的核心主题是否在每一卷中得到推进？
- 结局是否回应了开篇提出的核心问题？
- 格局是否有层递（个人→群体→世界→理念）？

## 质量量化指标

### 硬性指标（双平台通用）

| 指标 | 标准 | 检测方式 |
|------|------|----------|
| CJK字符数/章 | ≥2000 CJK汉字 | PowerShell脚本统计 |
| "不是X是Y"模式 | ≤2个/章（**头号AI指纹，663→31的教训**） | 正则: `/不是[^—]{2,30}——是/g` |
| 正文元话语 | 0次（禁止"卷一""第X章""前文""后文""本章"） | 人工扫描 |
| 模板段落 | 0个/单元 | 跨章文本相似度检测 |
| 章节差异性 | 每章有≥1个独特场景/对话/意象 | 人工审核 |
| 重复形容词 | 同词≤3次/章 | 词频统计 |
| 段落长度 | 主要段落≤200字 | 人工检查（手机阅读友好） |
| 破折号 | ≤20个/章（超标用脚本批量替换为中文逗号） | 自动化检测+替换 |
| AI口水词 | 0次/章：缓缓/不由得/眼底闪过/心中升起/说不出的/这意味着 | 关键词扫描 |
| 感叹号 | 战斗章≥2个/章 | 初稿检测 |
| 场景连续性 | 同一场景≤3章（连续3章以上必须切换） | 人工扫描 |
| 长线钩子间隔 | 同一长线钩子间隔≥2章 | 最近5章结尾检查 |
| 角色语音区分 | 去掉名字仍能分辨谁在说话 | 盲测 |

### 软性指标

| 指标 | 标准 | 评估方式 |
|------|------|----------|
| 人物声音区分度 | 对话中去掉名字仍能分辨谁在说话 | 盲测 |
| 情感场景冲击力 | 每单元≥3个"名场面" | 读者模拟 |
| 信息密度 | 无"跳过5章不影响理解"的废章 | 删除测试 |
| 钩子有效性 | 每章结尾让人想继续翻 | 逐章评估 |
| 结尾满意度 | 结局不引起"烂尾"评价 | 多轮审查 |

## 改良工作流

### 阶段一：全局诊断（2-3轮）

**目标**：找出制约评分的根本性问题，而不是头痛医头。

1. **通读评估**：找3-5章代表性章节（开头、中段、高潮、过渡、结尾各1章），进行十维评分
2. **结构审查**：画出全书的情绪曲线、爽点分布图、人物出场频率图
3. **问题定位**：识别全局性问题（如"中段全部疲软"）vs 局部问题（如"第37章人物崩坏"）
4. **优先级排序**：结构性>人物>节奏>语言>细节

**输出**：诊断报告 + 改良方案（先改什么、后改什么、预计效果）

### 阶段二：结构性改写

**这是9.0+和8.0+的真正分水岭。** 大多数小说改不到9.0+是因为只改了表面没有改结构。

1. **节奏重排**：如果中段疲软，需要砍章、合并章、新增高能章
2. **人物弧线修复**：如果人物崩坏，需要回溯到崩坏点，从根上修复
3. **爽点再分布**：确保每3-5章有一个"小爽"，每单元有一个"大爽"
4. **伏笔系统重审**：每条伏笔必须可追踪（编号、投放章、暗示章、回收章）
5. **高潮重设计**：每个单元的高潮是否足够强？不够需要重写

**关键原则**：
- 大胆砍：对废章不要心软，每砍一章都是在提升完读率
- 大胆加：在情绪低迷区插入高能场景
- 大胆调：章节顺序不是神圣不可改的

### 阶段三：逐章改写

针对每一章进行四个层面的改写：

#### 3A. 去AI味（通用铁律同步）

以下规则来自《黑龙醒》663次→31次清洗的实战教训，对任何改良操作都是第一优先级：

**头号目标："不是X，是Y"句式**：
- 全书最大的AI指纹。从第一章开始控制，不要等到积累了几百次才回头改
- 对话中的"不是"可保留（口语自然），叙事中的必须砍
- 如果必须对比，用"——"或另起一句

**AI口水词清单（发现=删除或改写）**：
- `缓缓`、`不由得`、`眼底闪过`、`心中升起`、`说不出的`、`这意味着`
- `他的眼神`、`她的眼眸`、`深邃`、`不可置信`

**元话语（发现=删除）**：
- "卷一""卷二""第X章""前文所述""后文再表""本章"
- 引用过去事件用具体内容描述，不用章节编号

**AI写作的其他特征和修复方法**：
- **并列式排比过多**：打破排比，改用不同句式
- **过度总结**：AI喜欢在段落末尾总结"这意味着……" → 删除总结，让读者自己感受
- **情感标签化**："他感到愤怒" → 用身体反应代替情感词："他握紧拳——指节在拳套里——咯吱咯吱"
- **模板化过渡**："夜深了……"、"与此同时……" → 每次用不同的过渡方式
- **抽象概念堆砌**："自由"、"选择"、"意义" → 用具体意象承载抽象概念
- **句式单一**：全部"主语——谓语——宾语" → 长短句交替，破句、断句、单字句
- **所有角色同一种语气**：检查每个角色的对话——去掉名字后还能分辨谁在说话吗？
- **感叹号和粗口不足**：战斗章的感叹号和角色的粗口应该在初稿就到位，后期注入效率极低且容易角色串音

#### 3B. 强化差异性
- 每章必须有**一个独特场景**（之前章节从未出现过的具体地点/时间/光线组合）
- 每章必须有**一个独特对话**（只有这一章这两个人会说出来的话）
- 每章必须有**一个独特意象**（只属于这一章的具体物件/声音/气味）

#### 3C. 提升情感冲击
- 用**具体动作**替代情绪描述
- 用**外部细节**反射内心状态
- 情感高潮前要有**沉默/停顿**——不是喊出来的高潮最有力
- "少即是多"——删掉所有解释性文字，只留事实

#### 3D. 优化钩子
钩子类型轮换使用：
- 悬念钩子："他不知道——这个决定——将在三天后——改变一切。"
- 画面钩子："路灯下——三个人的影子——叠成一片。"
- 金句钩子：一个让人记住的句子
- 情感钩子：一个让人心疼的细节
- 反转钩子：刚刚揭示的新信息
- 留白钩子：一个不解答的问题

### 阶段四：质量验证

对改写后的每一章进行自动化+人工验证：

```
自动化检测：
1. CJK字符数统计
2. "不是X是Y"模式计数
3. 跨章相似段落检测
4. 高频词统计

人工验证（模拟番茄读者）：
1. 3秒扫描测试：3秒内扫过一章，能否抓住核心事件？
2. 跳读测试：跳过3章再读，还能跟上吗？
3. 情绪测试：读完后有情绪波动吗？（爽/痛/燃/暖）
4. 记忆测试：读完一个单元后能回忆起几个名场面？
5. 钩子测试：每章结尾想翻下一章吗？
```

### 阶段五：读者模拟评审

以番茄老读者视角，对每个单元进行评审：

1. **第一遍**：快速浏览（模拟真实阅读速度）→ 打分（直觉分）
2. **第二遍**：仔细阅读 → 十维逐项打分 → 找出扣分项
3. **第三遍**：对照找出的问题 → 改写 → 自检

模拟评审自问清单：
- 如果我是付费读者，这章我愿意花多少钱？
- 这章有让我截图/划线/分享到书友群的内容吗？
- 这章结束后我有"再来一章"的冲动吗？
- 这一章让主角变得更让人喜欢还是更让人讨厌？
- 如果隔壁书架有一本同类型但评分更高的书，我为什么选这本？

### 阶段六：终审与微调

1. **全局一致性审查**：通读全书，检查人物一致性、伏笔回收、情绪曲线
2. **开头重审**：全书改完后，开头3章是否需要重新调整（因为后续内容已经变了）
3. **结尾重审**：结尾是否回应了开篇的核心问题？是否给了读者满意的"告别"？
4. **标题优化**：每章标题是否吸引点击？（番茄的章标题直接影响点击率）
5. **简介/文案优化**：如果提供简介，是否需要根据改后的全书重新写？

## 平台特有优化

### 番茄平台优化

#### 章标题优化
- 番茄读者先看标题再看内容，标题要有信息量
- 好标题："第X章 核心人物·核心事件"
- 高潮章标题要有分量感：单字标题=大事件

#### 每章开头优化
- 番茄的推荐流中，用户能看到前3-5行
- 每章开头必须在3行内抓住注意力
- 避免：景物描写开头、天气开头、哲学思考开头
- 推荐：动作开头、对话开头、悬念开头、反常现象开头

#### 段落长度控制
- 手机屏幕约显示15-20行
- 一段文字超过手机一屏=阅读疲劳
- 目标：主要段落不超过200字，穿插短段落（1-2句）

#### 番茄变现优化
- 完读率是核心——读者读完的比例越高，算法推得越多
- 前3章是生死线——完读率低于35%立刻优化
- 第7/21/45章是追更衰减节点——必须埋爆点挽留
- 章章结尾炸=更高的章节连续点击率=更多广告曝光
- 听书模式适配：对话像真的在说话、段落不超100字、避免大段纯设定

### 起点平台优化

#### 信息增量检查
- 每章读完后，读者"知道了什么新东西"？
- 改良重点：确保每章有至少一样信息增量（设定/人物/伏笔/关系推进）

#### 历史/设定硬伤扫描
- 起点读者会考据。涉及真实历史的内容需要有据可查
- 架空设定的规则前后一致——数字（时间/距离/数量）不能自相矛盾

#### 群像深度检查
- 每个配角的独立瞬间是否在最近10章中出现？
- 有没有配角连续10章沦为工具人？

#### 订阅留存优化
- 每3-5章必须有一个"值得付费"的瞬间
- 结尾钩子类型必须轮换，不能连续重复
- 上架章必须在情绪上升期（高潮刚开始时上架，不能在高潮前断）

#### 起点付费设计
- 免费期最后1章的结尾：三重钩子
- 付费章的信息密度不能低于免费章
- 付费章结尾：读者必须面临"不确定"——只能订阅下一章才能知道

## 常见问题速查

### 问题：中段疲软（完读率杀手#1）
**症状**：章节100-300之间，情节重复、冲突降级、读者流失
**修复**：
1. 砍掉所有"过渡章"——它们不需要存在
2. 在大情节之间插入"人物关系深化"场景
3. 引入新变量（新人物/新规则/新威胁）打破惯性
4. 在100、150、200、250章设置"中期高潮"

### 问题：人物工具化
**症状**：配角只出现在需要推动情节的时候，没有自己的生活
**修复**：
1. 给每个主要配角写一个"不在主角视线内时我在做什么"的暗线
2. 配角的行动必须由其自身动机驱动，不是"帮主角完成任务"
3. 配角在剧情不需要他们的时候也应该"存在"

### 问题：爽点模式化
**症状**：每次爽都是同一种模式（打败对手→获得认可→升级）
**修复**：
1. 爽点类型多样化：认知爽（发现真相）、情感爽（和解/告别）、智识爽（策略成功）、选择爽（艰难抉择后的自我肯定）
2. 爽的幅度变化：不是每次都大爽，小爽中爽穿插
3. 爽的方向变化：有时候"放下"比"得到"更爽

### 问题：结尾乏力
**症状**：前面的铺垫很好，但最后30章突然加速草草收场
**修复**：
1. 结尾需要至少一个完整单元的篇幅来"降落"
2. 每条主要伏笔必须回收
3. 主要人物都需要"告别场景"
4. 最后一章不是"事件结束"而是"情感收束"

### 问题：AI味重
**症状**：读者评论"像AI写的"、"模板化"、"没灵魂"
**修复**：
1. 删除所有总结性、解释性语句（AI的标志性特征）
2. 打破排比句式（AI最喜欢排比）
3. 加入不规则表达：破句、重复、口语化插入
4. 加入具体的、有颗粒感的细节（AI生成不了真实的细节）
5. 人物对话中加入口癖、地域特征、身份标记
6. 让叙述者的声音有态度（不中立、不客观、有情绪）

## 执行原则

1. **大胆改**：9.0+需要的不只是修修补补，是大胆的结构性改良。不敢动大结构=上不了9.0。
2. **先诊断后动手**：不要在没搞清楚根本问题之前就开始改文字。改了100章的错别字但没修结构性疲软=白改。
3. **每轮验证**：每改完一个单元，立即做十维评分+读者模拟。在问题出现时发现，而不是改完全书后。
4. **数据驱动**：能用数据说话就用数据说话。CJK统计、模式计数、相似度检测——这些比直觉更可靠。
5. **读者视角**：永远站在读者那边想问题。作者觉得重要的≠读者觉得重要的。
6. **改完≠改好**：第一次改写通常只解决表层问题。需要多轮迭代才能达到9.0+。
7. **保持作者声音**：改良的目标是让作者的声音更清晰、更有力，而不是用另一个声音替换它。

## 与作者协作

- 在结构性改动前，必须向作者解释"为什么改"和"改了之后的效果"
- 提供改前改后对比（具体到段落级别）
- 对于可能引起争议的改动（如砍章），先征求意见
- 每次评审后给出分数变化，让作者看到进展
- 记录所有改动，方便回溯和撤销

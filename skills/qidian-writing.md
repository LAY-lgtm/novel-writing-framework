# 起点中文网写作技能（Qidian Writing Skill）

## 元信息
- **平台**：起点中文网 / 起点读书（阅文集团）
- **适用类型**：东方玄幻、仙侠、都市、历史、科幻、悬疑——以中长篇（200章+）为主要目标
- **核心定位**：慢热可以做、设定必须硬、人物必须真、每章必须有信息增量
- **基于**：《黑龙醒》前100章实战 + writing-novel 阶段十三/十四 + 起点读者行为分析
- **与番茄平台的关系**：互补技能。写作前必须确认目标平台，两套方法论共用技术基础设施（CJK检测、破折号控制等），但节奏模型和内容策略完全不同
- **版本**：v1.1（2026-05-31）——首次创建（v1.0）+ 融合《黑龙醒》CLAUDE.md 十条铁律完整记录、三章一轮制度、场景多样性检查、长线钩子管理、元话语禁令、角色语音卡概念、项目配置模板（v1.1）

## 第一部分：起点 vs 番茄——元差异（写作前必读）

**核心引语**："这是起点爆款，不要按照番茄来。"——用户纠正，《黑龙醒》49章大修记录

| 维度 | 番茄爆款 | 起点爆款 |
|------|---------|---------|
| 读者耐心 | 3秒划走，前3句不抓人就没了 | 愿意读慢热设定，可以接受铺垫 |
| 爽点密度 | 3章一爽，不能断节奏 | 可以10章逐步铺垫，爽点要有层次 |
| 文风 | 短句/直给/动词轰炸 | 可以有一定文学质感，但禁止掉书袋 |
| 主角 | 全程碾压，不能弱 | 允许成长弧线、恐惧、犯错——但要最终站起来 |
| 群像 | 主角视角为主，配角工具化可以接受 | 群像可以充分展开，配角必须有独立瞬间 |
| 历史/设定 | 简化——读者不会考据 | 可以厚重、有考据感——但有考据感的读者也会考据你 |
| 结尾钩子 | 每章必须炸——悬念/危机/反转 | 悬念可层层递进——不要求每章炸，但要求每章让读者"还想往下看" |
| 情感模式 | 撕裂式爆发为主，安静的温馨极少 | 安静的感动也可以成立——但必须有情绪弧线的完整性 |
| 更新节奏 | 日更4000+字是基础 | 日更6000-10000字，断更掉收藏 |
| 生命周期 | 免费期→上架→持续爆更，吃前3个月 | 免费期养口碑→上架爆发→长期订阅，可以吃半年到一年 |

**共同底线（不可降级）**：
- 不能水、不能模板化、不能AI味
- 每章必须有钩子（类型可以不同，但必须有）
- 角色必须有真人的质感（不管什么平台）
- 反派必须有逻辑和动机

**最重要的一个认知差异**：
- 番茄读者的路径是：刷到→被前三句抓住→一口气读→爽了→追→不爽→划走。**情绪即流量。**
- 起点读者的路径是：搜到/被推荐→试读前20章试水→觉得"有点意思"→收藏养肥→上架后订阅→长期追。**信任即流量。**

这意味着：番茄靠**单章爆发**赢，起点靠**持续品质**赢。在起点，一节水了可能不会立即掉收藏，但会让你在"养肥期"被读者抛弃——你根本不知道他们是什么时候走的。

## 第二部分：起点读者画像

### 读者特征
- **年龄**：18-35岁为主，男性占70%+（东方玄幻/仙侠），女性读者在都市/言情品类显著
- **付费意愿**：强。起点读者是中国网文平台中付费习惯最好的群体
- **行为模式**：
  - "试水"阶段：免费期读20-50章，判断是否值得收藏
  - "养肥"阶段：收藏后每周扫一眼更新量，攒够一波再宰
  - "追更"阶段：上架后每天订阅，评论区活跃
  - "弃书"阶段：连续几章节奏不对→不声不响地走，**不会告诉你为什么**

### 读者核心诉求（按优先级）
1. **世界是自洽的**——设定不能前后矛盾。起点读者会截图存档、做考据、在评论区挑逻辑漏洞。如果你的秦朝背景出现了汉朝才有的官制，会有人发现。
2. **人物是真的**——主角的恐惧、犹豫、成长弧线必须真实。配角的独立存在感必须被感受到。起点读者对"纸片人"的容忍度极低。
3. **每章读完有"得到"**——不一定是爽，可以是：知道了一个新设定、理解了一个人物关系、发现了一个伏笔——但必须有所得。
4. **长期情绪有起伏**——可以有安静章节，但不能连续多章"没什么发生"。
5. **付费值得**——起点读者是真金白银付钱的，每章读完如果没有"还想看下一章"的冲动，就可能在下一章停止订阅。

### 起点平台特有的读者行为（务必了解）
- **章评（本章说）文化**：读者会在段落间发评论，互相讨论。这是双刃剑——好的章评能拉动气氛带更多订阅，差的章评（"这章水"）会劝退潜在付费读者
- **推荐票/月票/打赏机制**：不是装饰。月票榜直接带来曝光，打赏带来粉丝值——粉丝值高的读者在评论区有更大话语权
- **养肥警告**：起点读者大量采用"养肥"策略——收藏后等几十章再宰。这意味着你更新的质量会延迟1-2个月才被"宰"到。一章水了，可能到两个月后才体现在掉订阅上——**滞后反馈是起点最大的写作陷阱**

## 第三部分：起点节奏模型

### 核心理念：信息增量驱动

番茄的节奏由"战斗/爽点"驱动。起点的节奏由**信息增量**驱动。

**什么是信息增量？**
每章读完后，读者知道了至少一样他之前不知道的东西：
- 一条新设定（世界的规则更清晰了）
- 一个人物的侧面（原来这个人还有这一面）
- 一个伏笔被埋下/回收（"原来XX和XX有关系"）
- 一次关系的推进或破裂（"他终于对谁说了那句话"）
- 一个谜团的出现或解答

**信息增量的反面**：
- 主角走了一段路，什么也没发现——没有信息增量
- 配角说了一段话，但说的都是读者已经知道的——没有信息增量
- 打了一场仗，但除了"赢了"没有揭示任何新东西——只有爽，没有增量。起点读者会在三四场之后开始疲劳。

### 起点三阶节奏

#### 免费期（前20-50章）：建立信任
- **目标**：让读者觉得"这本书有点东西，值得养"
- **每章要求**：
  - 必须有信息增量（世界观每一章都展开一点点）
  - 主角必须有主动决策（展示性格，建立读者认同）
  - 钩子不一定要炸，但要让读者"想知道接下来会发生什么"
- **前5章的硬指标**：
  - 第1章：主角的"异常"被展示（他有什么特别之处？他面临什么问题？）
  - 第3章：世界观的第一层展示（这个世界怎么运作？有什么规则？）
  - 第5章：第一个小高潮或悬念确立（让读者决定要不要继续）
- **节奏密度**：可以比番茄慢一倍。番茄要求3章一爽，起点免费期可以5-10章铺垫一个大爽点

#### 上架期（上架前后10章）：爆发期
- **目标**：让免费读者付费订阅，保住首订数据
- **关键节点**：
  - 上架前3章：铺垫高潮（读者的"我想看"达到峰值）
  - 上架章：必须在情绪上升期——**不能在高潮前断，要在高潮刚开始时上架**
  - 上架后3章：高潮+余波+新勾子（让付了费的读者觉得值，然后继续订阅）
- **上架章硬指标**：
  - 必须包含至少一个让人"爽"的瞬间
  - 必须埋下一个更大的悬念（让人知道后面还有更精彩的）
  - 结尾必须是强钩子——"不订阅下一章就亏了"

#### 稳定期（50章以后）：长期维护
- **目标**：维持订阅，防止养肥流失
- **节奏**：
  - 战斗章 + 信息章 + 过渡章的轮换可以有弹性
  - 不是每天都要炸，但**每3-5章必须有一个信息爆点或情绪高点**
  - 过渡章可以长达2-3章，但必须有信息增量支撑
- **危险信号**（意味着读者正在流失）：
  - 连续3章没有新增设定/新人物/新矛盾
  - 连续2章主角没有主动决策
  - 反派连续N章没有造成实质性威胁
  - 所有配角都退为背景

### 节奏检查清单（每10章一次）
- [ ] 读者在这10章里"知道"了至少3件新的事吗？（世界观/人物/伏笔各至少1件）
- [ ] 有没有一章读完后读者可以说"这章什么都没发生"？
- [ ] 免费期的节奏是否在"太慢→无感"和"太快→没有厚重感"之间找到了平衡？
- [ ] 最新一章的结尾，读者有多想点"下一章"？如果不想——为什么？

## 第四部分：起点质量自检清单

**每次写完一章后执行，满分10分。低于6分的章必须重写。**

### 1. 历史/设定硬伤扫描
- [ ] 涉及真实历史的内容是否有据可查？（官制、地理、器物、称谓）
- [ ] 架空设定是否前后一致？有没有"第10章说XX只能三天一次，第50章主角一天用了三次"？
- [ ] 数字是否合理？（时间线、距离、数量——起点读者会算）

### 2. 逻辑闭环
- [ ] 本章的所有能力和设定是否在之前埋过伏笔？
- [ ] 敌人为什么这么强/这么弱——有合理的理由吗？
- [ ] 主角的决策逻辑——读者能理解他为什么这么做吗？（不一定要同意，但要能理解）
- [ ] 有没有"剧情需要"导致的逻辑跳跃？（敌人突然变傻、环境突然配合）

### 3. 人物驱动
- [ ] 主角在本章中至少有1个主动决策（*不是等人汇报后做判断，是自己去找答案/做选择*）
- [ ] 主角的决策是基于他的性格和处境做出来的，不是"剧情需要这个决策"
- [ ] 关键配角在本章中的行为符合他们自己的动机线（*不是被作者操控的工具人*）

### 4. 爽点高级度（起点特有维度）
起点读者的"爽"比番茄更挑剔。爽点分五级：
1. 简单击杀（基础，不能全靠这个）
2. 实力突破/升级（有铺垫的情况下效果翻倍）
3. **信息差碾压**：主角知道别人不知道的事，利用信息优势翻盘——起点读者的最爱
4. **规则反转**：发现了规则的漏洞/隐藏条款，从被规则压制变成利用规则
5. **真相揭示**：一个长期伏笔被回收，"原来是这样"——高级爽点，起点读者会为此月票打赏

**要求**：每10章中至少包含3/4/5中的一种爽点。长期只有1和2，会被评价为"爽是爽了，但没意思"。

### 5. 钩子多样性
- [ ] 本章结尾钩子类型（必须标注）：
  - A类-悬念："他推开门的瞬间，看到了不该出现的东西。"
  - B类-危机："脚步声越来越近，他来不及跑了。"
  - C类-反转："那个人转过头来——是三天前就死了的人。"
  - D类-信息揭露："木牍的最后一行写着他的名字。"
  - E类-情感："他终于说出口了，那句话说出口的瞬间，所有人都安静了。"
- [ ] 连续3章的钩子类型不能重复
- [ ] 本章钩子是否足够强让读者"必须看下一章"？（*0-10分自评*）

### 6. 反派深度
- [ ] 本批次中是否出现了有逻辑、能对话的反派？
- [ ] 反派的行为有"他自己的道理"吗？（*不一定是正确的道理，但必须是一个真人能说出口的道理*）
- [ ] 主角和反派之间有过真正的对话博弈吗？（*不只是打架前的放狠话*）

### 7. 配角独立度（起点特有维度）
- [ ] 每个有名有姓的配角，在最近10章中有过至少1个独立的、让人记住的瞬间吗？
- [ ] 这个瞬间不一定要和主角有关——可以是他自己的决定、他自己的恐惧、他自己的闪光
- [ ] 有没有配角在本章中沦为"工具人"（纯粹为了递消息/挡刀/凑人数而出场）？

### 8. 主动战斗（主角）
- [ ] 每10章中主角至少有1次亲手参与战斗（*不是放技能远程轰炸，是亲手——拳头、武器、身体接触*）
- [ ] 他在战斗中有恐惧吗？手指发抖但还捅进去了——比"冷静地捅进去"更好

### 9. AI指纹扫描
**必须扫描以下关键词/句式：**
- [ ] "不是……而是……"句型：本章出现_____次（目标≤2/章）。**这是全书最大的AI指纹——《黑龙醒》从663次清洗到31次，耗费了数十轮交互，是所有修复中耗时最长的单项。**
- [ ] 砍掉以下AI口水词：`缓缓`、`不由得`、`眼底闪过`、`心中升起`、`说不出的`、`这意味着`、`深邃`、`不可置信`
- [ ] "他的眼神" / "她的眼眸"：每2000字出现不超过1次
- [ ] 对话中是否出现"XX道："（正常），但不能连续6句对白用同一句式
- [ ] 段落首句是否有连续3段以上用"某人+动词"结构？
- [ ] 是否有AI惯用的总结句？（如"这不仅仅是一场战斗，更是一次…"）
- [ ] 检查感叹号数量：战斗章≥2个/章；粗口是否按角色语音卡到位

**铁律：不达标的当场修，不要攒到51章一起修。每写完3章立即跑AI指纹扫描——写作时自觉避免"不是X是Y"句式，对话中的"不是"可以保留，叙事中的必须砍。**

### 10. 情绪曲线
- [ ] 最近3章的情绪分布（每章标注一种主情绪）：爽 / 紧张 / 重（代价感）/ 好奇 / 温馨 / 愤怒
- [ ] 3章内至少覆盖：爽×1 + 紧张×1 + 重×1
- [ ] 有没有连续3章只有"温馨"？——有的话必须打破
- [ ] 本章的情绪高点在哪里？（*标注行号*）

## 第五部分：群像写作铁律（起点特化）

**核心引语**："起点读者追群像文的核心心理是'这群人在一起才无敌'——如果只有主角一个亮眼，队伍就变成了跟班。"——55章实战教训

### 配角饱满度三层模型

#### 第一层：独立高光
- 每10章中，每个主要配角必须有至少1个**独立的、不依附于主角的**高光瞬间
- 高光不限于战斗力——可以是：
  - 技能高光：他擅长的那个技能在关键时刻救了所有人
  - 情感高光：他说了一句只有他这个性格的人会说的话，让所有人沉默了
  - 物件高光：他身上那件一直不起眼的物件，终于展示了它的用途
  - 沉默高光：所有人都慌了的时候，他没有慌。他的沉默本身就是高光。
  - 身体高光：他用身体挡了什么/承受了什么，没有喊疼

#### 第二层：低谷/弱点
- 每个主要配角必须有至少1个被展示出来的弱点或低谷时刻
- 完美配角 = 无聊配角 = 读者不关心的配角
- 弱点可以是什么：恐惧某种东西、控制不住的冲动、过不去的往事、说不出口的话

#### 第三层：独立故事碎片
- 每个主要配角在每20章中至少有1个"和你没关系但我想知道"的独立故事碎片
- 不需要完整展开——一个细节就够了：他半夜去了哪里？他桌上多了一样什么东西？他在无人时做了什么？
- **目的**：让读者觉得"这个角色有自己的生活"，而不是"这个角色只有主角在的时候才存在"

### 配角分配硬指标（每10章）
- 主角高光章 ≤ 5/10章（*另外5章给配角发光*）
- 每个主要配角在10章内至少1个独立瞬间
- 每场战斗/行动的核心参与者 ≤ 5人（*其他人退为背景支援*）
- 可有可无的角色合并（*如：两个功能相似的配角只保留一个，把两者的特点合并到他身上*）

### 群像检查（每10章一次）
- [ ] 列出每个主要配角在最近10章中的独立瞬间
- [ ] 有没有配角在最近10章中完全沦为工具人？
- [ ] 读者现在能说出每个主要配角的名字+特征+心里在想什么吗？
- [ ] 如果删掉某一个配角，这10章会有什么不同吗？如果不会——这个角色没有存在的必要

## 第六部分：付费章节策略

### 上架时机判断
- **不能上架太晚**：免费章太多→读者被惯坏了→付费转化率低
- **不能上架太早**：还没建立信任→付费意愿不足→首订惨淡
- **最佳时机**：第一个大高潮的**正在进行时**。读者正在兴奋→需要付费才能继续→转化率最高

### 上架前准备清单
- [ ] 免费期最后一章的结尾：三重钩子（悬念+危机+下一章的暗示）
- [ ] 上架章的标题已经设计好——标题本身就是一个钩子
- [ ] 上架当天的更新量：至少3章（让首订数据好看，给读者"一次性爽够"的感觉）
- [ ] 上架感言已写好——真诚、有个人特色、承诺更新计划

### 付费章节设计原则（每章）
- 结尾优先：**先设计结尾钩子，再往前写整章。** 结尾决定了读者会不会订阅下一章
- 每章结束时读者必须面临一个"不确定"——你不知道下一章会怎样，所以你要订阅
- "每章结尾要让读者愿意付费看下一章"——这是起点的核心商业模式，不是修辞要求
- 断章不能在高潮最高点（会被骂"断章狗"——但适度的骂也是一种互动），也不能在平淡无奇处（没人会订阅下一章）。**在高潮的上升期断**，读者窝着火也必须点下一章。

### 订阅维护策略
- **每3-5章必须有一个"值得"的瞬间**：让读者觉得"这章没白订"
- **爆更后必须有质量**：一顿爆更10章但信息密度太低，反而会引发"水"的评价
- **评论区互动**：定时看评论，对合理建议做出回应。起点读者看得到你——一个活着的、有诚意的作者

## 第七部分：起点写作流程

### 核心制度：三章一轮（铁律，不可跳过）

**每写完三章为一轮，立即停止写作进入自检。** 这是从《黑龙醒》050-100章修复中买来的最昂贵的教训——不要攒到51章一起修。

**为什么是三章而不是每章或每10章？**
- 每章自检太碎，打断写作流
- 每10章自检太晚，问题积压——663次"不是X是Y"如果每3章扫一次，第一轮就会发现异常
- 三章是一个"可见趋势"的最小单元：够看出钩子是否在重复、情绪是否有起伏、反派是否有进展

**三章一轮执行流程**：
1. 写完3章初稿
2. 立即跑两轮自检：**先技术自检，后起点质量自检**
3. 自检不达标 → 当场修复 → 跑AI指纹扫描 → 确认通过
4. 每10章（即3轮+1章）做一次批次审查
5. 自检通过后生成Word文档供人工检查

### 动笔前的平台确认（第一步，不可跳过）
1. 确认目标平台：**起点**。如果是番茄，切换到 piqie-writing 或 writing-novel（阶段一至十二）。
2. 确认品类：东方玄幻/仙侠/都市/历史/悬疑——不同的品类在起点有不同的读者预期
3. 确认总章数目标（起点中长篇基准：200章+）

### 项目配置初始化

在动笔前，为小说项目创建以下基础设施（参考《黑龙醒》项目结构模板，见第十一部分）：

**必须创建的文件：**
- `CLAUDE.md`：项目配置+自检清单（本技能提供模板）
- `故事圣经.md`：核心设定的唯一真实来源——所有设定写在这里，写正文时引用这里，禁止正文中临时改设定
- `大纲.md`：逐章规划（起点版需要比番茄更厚的设定层）
- `人物档案/`目录：每个重要角色一个独立文件
- `伏笔记录.md`：已埋+待回收伏笔的管理表
- `时间线.md`：故事内事件时间线，防止时间矛盾

**配角档案规则**：每出现一个新配角（无论戏份多少），立即创建人物档案文件。不要等——等到后期你会忘记这个配角的细节。

**角色语音卡**：为主要角色（主角+2-3个核心配角）创建语音卡文件，定义：
- 口头禅/高频词
- 粗口风格（如果有）
- 说话节奏（短句/长句/喜欢用哪种句式）
- **目的**：每次写这个角色的对话时，打开语音卡对照——确保每个人说话的方式不一样

### 大纲设计（起点版）
起点的大纲需要比番茄更厚的"设定层"：
- 世界观设定文档（力量体系、规则、历史背景——这些读者会考据的）
- 人物关系图（每个主要配角的动机线和成长弧）
- 伏笔管理表（何时埋、何时收、几条线并行）
- 分卷规划（每卷的核心冲突和收束方式）
- 第一个大高潮的位置（决定上架时机，见第六部分）

### 每章写作流程
1. **写前确认**：这一章在这个位置应该提供什么信息增量？
2. **初稿**：先写结尾钩子，再往前写正文。**初稿就要把感叹号和粗口放到位**——后期注入效率极低。
3. **四维自检**（技术层）：
   - CJK字数 ≥ 2000
   - 全角破折号（U+2014）≤ 20，**超标立即用脚本批量替换为中文逗号——不要手工处理**
   - 开头80 CJK内必须有动作动词
   - 结尾必须有钩子（标注钩子类型）
4. **起点质量自检**（对照第四部分的10项清单）
5. **AI指纹扫描**：扫描"不是…是…"、AI口水词（缓缓/不由得/眼底闪过/心中升起/说不出的/这意味着）、模板句式
6. **修复不合格项**——**不达标的当场修，不要攒到后面一起修**
7. **写入文件**
8. **每出现新配角 → 立即创建人物档案**

### 场景多样性检查（加入四维自检）

**《黑龙醒》084-100章连续17章在长城垛口——读者闭眼就是同一画面。** 这是血的教训。

- [ ] 最近连续几章在同一场景？（核心场景）→ 连续3章以上在同一场景必须切换
- [ ] 场景轮换池是否在用？（列出5+个可用场景：城内/城外/林线/沟底/村寨/地宫/营房/路上）
- [ ] 每次场景切换是否可以顺带插入留守组/其他POV，保持双线/多线叙事？

### 长线钩子管理

**"铁水还没烧"作为长线悬念不能每章结尾都提。** ——《黑龙醒》097-099教训

- 长线倒计时/悬念每隔3-4章提一次就够
- 中间用其他类型的钩子（人物钩子、情感钩子、即时危机）
- 如果连续3章结尾都在提同一个长线悬念 → 读者麻木，变成空炮
- 检查：列出最近5章的结尾钩子，看长线钩子的间隔是否≥2章

### 每10章批次审查（起点版）
除了通用的四维自检外，增加的起点专项：
- [ ] 信息增量轨迹：这10章中读者"知道"了哪些新东西？
- [ ] 爽点高级度分布：有没有3/4/5级爽点？
- [ ] 配角独立度：每个主要配角有独立瞬间吗？
- [ ] 钩子类型轮换：连续3章无重复；长线钩子间隔≥2章
- [ ] 反派进展：新威胁出现了吗？反派升级了吗？
- [ ] 牺牲检查：这10-15章中有人死了吗？（不是每批都要死，但不能长期不死）
- [ ] 主角亲手战斗：这10章中主角至少有1次亲手参与战斗吗？
- [ ] 情绪曲线：10章整体是否覆盖了"爽+紧张+重+好奇"四个维度
- [ ] 场景多样性：有没有连续5章以上钉死在同一场景？
- [ ] 破折号批量检查：超标章节是否已用脚本自动替换？

### 每卷（约50-80章）全卷审查
**全写完后再做全卷一致性审查。** 不要在写作中途做全卷审查——会打乱节奏。

- [ ] **历史硬伤全卷扫描**：有没有前后矛盾的时间线、距离、数量？
- [ ] **伏笔盘点**：埋了多少条线？收了哪些？哪些已经拖太久没收了？
- [ ] **人物弧线检查**：主角的成长是否线性（从恐惧到果断、从旁观到执行）？配角的弧线是否被忽略？
- [ ] **牺牲检查**：这一卷中有人付出代价了吗？（不一定是死，但必须有可见的代价——血、伤疤、骸骨、文字记录）
- [ ] **情绪曲线总览**：整卷有没有情绪"死区"——连续超过10章情绪没有起伏？
- [ ] **连续"无所得"警示**：有没有连续3章读者读完说不出"我知道了什么新东西"？
- [ ] **全卷人称/名字一致性**：人名在一卷中前后一致吗？（手动检查+脚本辅助）
- [ ] **配角存在感盘点**：列出每个有名配角的出场次数和独立瞬间数量——有没有人被遗忘了超过20章？

## 第八部分：禁止列表（起点版）

### 平台认知禁则
1. **禁止把起点的慢热理解成"可以水"**：慢热是有信息增量的慢慢展开；水是读者读完什么都得不到。起点读者永远不会原谅水。
2. **禁止低估起点读者的考据能力**：你写的每一个历史细节、数字、距离，都会有人验证。
3. **禁止把群像展开理解成"每个人都写一段"**：群像是有重点的展开——不是轮流出场打酱油。
4. **禁止把"安静的感动"当成默认可用的情感模式**：可以用，但不能是唯一的情感底色。依然需要撕裂式的爆发作为对比。

### 写作模式禁则
5. **禁止写纯日常模板**：和番茄一样——没有冲突就没有故事。
6. **禁止主角变成旁观者**：起点允许主角弱，不允许主角没有行动力。弱但要去做——这是起点读者的萌点。弱且不做——这是弃书理由。
7. **禁止所有冲突靠"观察"和"解读"解决**：必须有人流血、有人受伤。
8. **禁止连续超过3章无信息增量**：不用每章都炸，但每章必须有"读者知道了新东西"。
9. **禁止把"继续"/"明天继续"当结尾**：起点读者为每章付费——"明天继续"是对付费读者的侮辱。
10. **禁止正文中出现元话语**："卷一""卷二""第X章""前文所述""后文再表""本章"——这些是作者视角的文字，读者看到立刻出戏。引用过去事件用具体内容描述，不用章节编号。
11. **禁止感叹号和粗口留到后期补**：初稿不放感情，后期加不回来。战斗章初稿就≥2个感叹号，会爆粗的角色的粗口初稿就写进去。后期注入效率极低且容易出错（角色串音）。
12. **禁止场景钉死在同一个地方**：连续3章以上在同一核心场景 → 必须切换。084-100连续17章长城垛口的教训。

### 付费禁则
10. **禁止高潮前断章**：上架章要在高潮刚启动时，不能在高潮前夜。
11. **禁止付费章的信息密度低于免费章**：读者花钱买更少的内容——这是最快的自杀方式。
12. **禁止承诺的更新量做不到**：起点读者的信任建立在更新数据上。说日更6000，连续三天更4000，就会有人弃。
13. **禁止回复评论时和读者对骂**：可以解释、可以感谢、可以沉默。不要吵架——起点评论区吵架劝退路人的效率是惊人的。
14. **禁止连续多章用同一种钩子**：即使信息增量有变化，结尾钩子类型重复3次以上，读者会产生"这人只会这一招"的疲劳感。

### 角色禁则
15. **禁止全书无正面角色死亡**：每个大篇章（约80章）必须有至少1次永久性牺牲。起点读者对"死不了"的群像没有紧张感。
16. **禁止反派全是不会说话的怪物**：必须有能辩论、有自己逻辑的人类反派。
17. **禁止配角沦为工具人超过10章**：如果某个配角已经连续10章只是"站在那里"，要么给他一个独立瞬间，要么在这批里让他消失一段时间。

### 世界观禁则
18. **设定临时改不得**：开篇定好的规则，到了后面不能"哦我之前说错了"或者"其实还有一个隐藏规则我没说"。起点读者会截图。
19. **禁止后期追加的核心设定超过前期基础**：前50章奠定的世界观不能在第300章被推翻重来。可以扩展，不能推翻。
20. **禁止数字自相矛盾**：金钱数额、军队人数、距离、时间——所有数字必须前后一致。

## 第十部分：实战经验记录（持续更新）

### 来自《黑龙醒》050-100章的十条血泪铁律（2026年5月）

**这十条铁律不应该存在。每一条都是因为犯了错才被写下来的。每次写作前必读。**

---

#### 铁律一：禁止AI句式——"不是X，是Y"是全书最大的AI指纹

- **事实**：050-100章原始版本中出现了663次"不是……而是……"句式。经过数十轮交互清洗，最终降到31次。**这是所有修复中工作量最大的单项。**
- **教训**：后期清洗的成本是指数级的——如果初稿就控制住了，根本不需要花数十轮交互。
- **执行规则**：
  - 不要用"不是A，是B"做判断对比。直接写B。
  - 如果必须对比，用"——"破折号或另起一句。
  - 每写完三章立即grep扫描，超过2次/章立刻回头改。
  - 对话中的"不是"可以保留（口语自然），叙事中的必须砍。
- **为什么这很重要**：读者可能说不出"AI味"是什么，但663次"不是X是Y"读下来，潜意识里会觉得"这本书不像人写的"。起点读者对AI味的容忍度远低于番茄——他们愿意读慢热设定，但不愿意读机器生成的文字。

---

#### 铁律二：感叹号和粗口必须在初稿中就到位

- **事实**：050-100章修复中，感叹号从6→35、粗口从1→23，全部是后期注入的。后期注入效率极低且容易出bug（"老秦腔"事件——把老秦的台词风格错误地复制到了其他角色身上）。
- **教训**：初稿不放感情，后期加不回来。感情不是"修饰"，是文字本身的气质。
- **执行规则**：
  - 战斗章每章≥2个感叹号（初稿就写出来，不要等后期加）
  - 每个会爆粗的角色在最近3章至少粗口一次（按角色语音卡来）
  - **角色语音卡定义了每个人的粗口风格——直接拿来用，不要自己造。**
- **反面教材**：想象一个老兵在战场上被箭射中肩膀，他说"啊，好疼。"——这不是人说的话。他说"日他先人！"——这才是。

---

#### 铁律三：正文中绝对禁止出现元话语

- **事实**：Ch56/66/67/70/76/91/94/100等多章中出现了"卷一""第二章""前文""后文""本章"等元话语，在修复中被一一清除。
- **教训**：元话语是作者/编辑视角的文字，读者看到会立刻出戏。起点的读者沉浸在故事里，你不应该从故事外伸一只手进去拍他的肩膀说"注意看这一章"。
- **执行规则**：
  - **绝对禁止在正文中出现**："卷一""卷二""第X章""前文所述""后文再表""本章"
  - 引用过去事件时，用具体内容描述，不用章节编号：
    - ❌ "如第三十七章所述，长城的裂缝已经开始扩大。"
    - ✅ "城寨那夜，裂缝还在三尺宽。现在能塞进一只手了。"
  - 章节序号/卷号只放在文件头和目录中，不进入正文
  - 如果需要在正文中标注时间/地点，用角色视角自然带出："他们到长城的第七天"——不是"第七章·第七天"

---

#### 铁律四：每10-15章必须有人死

- **事实**：050-100章长达51章零正面角色死亡。直到096章老兵才死——而且是三轮修复中最后才加入的。老兵死后，全书的"牺牲爽"才算从零补回了一笔。
- **教训**：没有牺牲，就没有代价。没有代价，就没有情感重量。读者不会为一群永远不死的人紧张。起点读者尤其如此——他们愿意付钱追更，但不是付钱看一场永远不死人的战争。
- **执行规则**：
  - 每10-15章至少一个读者认识的人死亡
  - 不需要每批都死——但100章只死1个太少了
  - **阵亡角色的名字要提前让读者熟悉**——老兵在092章就给过主角一颗石子，读者对他有印象后再让他死，冲击力完全不同
  - 如果某批真的没有死人，必须有其他形式的可见代价（重伤留疤、失去某种能力、重要物件损毁）
- **反面教材**：如果到了第80章还没有一个正面角色死亡，读者会潜意识里觉得"这本书里好人不会死"——然后所有战斗都失去了紧张感。

---

#### 铁律五：主角必须亲手战斗

- **事实**：050-100章原始版本中，主角从"手指发抖但还捅进去"的执行者退化成了"站在垛口上感应/调度/标标记线"的指挥官。096章才加入第一次亲手杀人的场景（秦剑捅进步卒喉咙），095章加入干草沟底反冲锋。
- **教训**：起点允许主角弱，不允许主角没有行动力。从"执行者"变成"观察者"不是成长，是角色退化。当主角最激烈的动作是"站在高处看数据"时，读者会对他失去兴趣。
- **执行规则**：
  - 每10章至少1次主角亲手战斗：不是放技能远程轰炸，不是调度指挥，是亲手拿武器捅/劈/刺/砸
  - 他有恐惧——怕了但还捅进去，比"冷静地捅进去"更有力
  - 如果某一章主角没有亲手战斗，他必须在这一章中有至少1个主动决策（自己去找答案，不是等人汇报后做判断）
- **为什么起点特别看重这个**：番茄读者可以接受主角"远程操控"——只要爽就行。起点读者追的是"这个人"的成长故事——如果他都不动手了，故事就失去了核心。

---

#### 铁律六：场景不能钉死在同一个地方

- **事实**：084-100章连续17章在长城垛口。读者闭眼就是同一画面——城墙、裂缝、敌人、铁水。095章干草沟底反冲锋才部分打破了视觉疲劳。
- **教训**：场景即画面。同一画面连续超过5章，读者会产生视觉惯性——开始跳读景物描写、开始模糊战斗差异。
- **执行规则**：
  - 连续3章以上在同一核心场景 → 必须切换
  - 维护一个场景轮换池（≥5个可用场景）：城内/城外/林线/沟底/村寨/地宫/营房/路上
  - 每次场景切换顺带插入留守组/其他POV，保持双线/多线叙事
  - 场景切换不需要大动作——主角出门、探子回城、远处发现异常——都可以自然切换

---

#### 铁律七：钩子不能连续重复（含长线悬念管理）

- **事实**：097/098/099连续三章结尾都是"铁水还没烧"——同一个长线悬念被连用三次。读者从"铁水什么时候烧？"变成"又是铁水，知道了知道了"。
- **教训**：长线倒计时作为钩子有其价值——但每章结尾都提就变成了噪音。长线钩子每隔3-4章提一次就够，中间用其他类型的钩子。
- **执行规则**：
  - 每章结尾钩子类型必须和前一章不同
  - 五类钩子轮换：悬念→危机预告→反转→疑问→温情收束
  - 长线倒计时/悬念每隔3-4章提一次——变成空炮读者就麻木了
  - 检查：列出最近5章的结尾钩子，长线钩子间隔必须≥2章

---

#### 铁律八：反派必须能说话

- **事实**：050-100章只有1个"能说话的人类反派"，且只出场了4章。另一个反派只露过两次面，第三个是持续性威胁但没有正面对话博弈。怪物类反派是非人反派，不能对话。
- **教训**：全是不会说话的怪物 → 战斗变成了"打怪" → 失去了人类冲突的深度。起点读者需要的是"我和你的道理不一样"的对抗，不是"我要打死你因为你是个怪物"。
- **执行规则**：
  - 每卷必须引入至少1个新的人类/人形反派，和主角有过正面语言交锋
  - 反派要有自己的逻辑和动机——不只是"来打XX的敌军"
  - 反派和主角之间必须有真正的对话博弈（不限于战场——可以是书信、使节、被俘后的对话）
  - 人类反派 ≠ 最终BOSS——他是整个反派谱系中的一环
- **反面教材**：如果反派连续50章全是裂缝生物、异兽、深渊怪物——读者会在某个时刻问："人呢？我想看人和人的对抗。"

---

#### 铁律九：破折号控制必须自动化——不要手工处理

- **事实**：每次重写/扩充都会导致破折号超标（严重时一章102个破折号）。修复了35章的破折号超标问题，反复修复极其耗时。
- **教训**：手工替换破折号是纯机械劳动，不应该由人来完成。脚本一次性批量搞定。
- **执行规则**：
  - 技术自检脚本中必须包含：破折号计数 → 超过20个 → 自动将超出部分替换为中文逗号
  - **不再手工处理破折号**——脚本批量解决
  - 建立破折号白名单：对话中的破折号（表示打断/转折）优先保留，叙事中的优先替换

---

#### 铁律十：去AI化要在初稿就做对

- **事实**："不是X是Y"从663→31花了数十轮交互，是所有修复中耗时最长的。感叹号和粗口的后期注入效率极低。
- **教训**：后期修复AI味的成本是初稿的10倍以上。不要存"后期再修"的侥幸心理。
- **执行规则**：
  - 每写完3章立即跑AI指纹扫描（不是每10章，不是每卷——是每3章）
  - 检测项：不是X是Y次数、感叹号数量、粗口数量、AI口水词密度
  - 不达标的当场修，**不要攒到51章一起修**
  - 三章一轮制度就是为这个铁律服务的

---

### 来自《黑龙醒》卷一全卷审稿的教训（2026年5月）
- 起点 vs 番茄的**8维差异表**（见第一部分）是本技能诞生的直接原因——如果不能在第一稿就区分平台，后期修正的工作量是灾难性的
- 群像配角的"独立高光"——如果不是有意识地分配给每个配角，主角会自然地吃掉所有高光。这是叙事引力——主角的存在感天然大于配角，你必须主动抵抗这个引力
- 代价必须"可见"——血、伤疤、骸骨、文字记录。抽象的代价对起点读者没有冲击力。"他很痛苦"不如"他手指上被矛杆磨掉的皮，到现在还没长回来"

## 附录A：起点章节尾部钩子类型库

**A类-悬念**：
- "他不知道的是，那个人已经等在那里三天了。"
- "屏幕上的数字还在跳动。没有人注意到，最后一个数字比其他的慢了半拍。"
- "信的最后一行，是他的名字。但他从来没有给这个人写过信。"

**B类-危机**：
- "门外的脚步声在三步外停住了。然后是钥匙插入锁孔的声音。"
- "他的手指已经开始透明了。从指尖开始，正在往上蔓延。"
- "屏幕上出现了倒计时。他还有四十秒。"

**C类-反转**：
- "那个人转过身来——是他在第一层就已经确认死亡的人。"
- "她全程都在说谎。他打开她的手机，看到了真正的消息记录。"
- "剑没有刺中。不是因为他躲开了。是有人替他挡住了。"

**D类-信息揭露**：
- "木牍上还有字。他把墨拓上去之后，看到了隐藏在原文下面的第二层文字。"
- "监控录像显示的不止这些。他把进度条往后拉了三十秒。"
- "DNA报告只有一行注释：样本与数据库中的第7号档案匹配。"

**E类-情感**：
- "她说这句话的时候没有哭。但他知道，她上一次哭是在十七年前。"
- "他终于说出口了。屋里没有人说话。窗外的风声替他做了回答。"
- "他跪下去的姿势和十年前一模一样。唯一的区别是这次面前没有人了。"

# 长篇小说写作Skill（基于500章实战）

## 核心教训

写500章小说最大的教训：**问题不在写的时候暴露，而在写完回头看的时候才看得见。** 本skill的目标是把回头看才能发现的问题，提前到写的阶段就预防。

---

## 多技能叠加规则（开写前必须确认）

本技能是写作技能体系的**基础层**。当你同时加载多个写作技能时，以下规则决定谁的优先级更高：

### 技能体系总览

```
writing-novel（基础层：通用流程+技术规范）
    │
    ├── piqie-writing（番茄平台层：快节奏爆款专项）
    │   └── 当选择番茄平台时，平台层规则覆盖基础层
    │
    ├── qidian-writing（起点平台层：慢热品质专项）
    │   └── 当选择起点平台时，平台层规则覆盖基础层
    │
    └── zhi-dou-writing（类型层：智斗/权谋专项）
        └── 当写智斗类型时，类型层规则叠加在平台层之上
```

### 加载顺序

1. **动笔前第一步：确认平台+类型**
   - 起点还是番茄？
   - 智斗类型还是通用类型？
   
2. **第二步：加载对应的技能组合**
   - 番茄 + 通用 = writing-novel + piqie-writing
   - 起点 + 通用 = writing-novel + qidian-writing
   - 番茄 + 智斗 = writing-novel + piqie-writing + zhi-dou-writing
   - 起点 + 智斗 = writing-novel + qidian-writing + zhi-dou-writing

3. **第三步：技能叠加优先级**
   - 类型层（zhi-dou-writing）> 平台层（piqie/qidian）> 基础层（writing-novel）
   - 当规则冲突时，更具体层的规则胜出

### 冲突解决速查表

| 冲突场景 | 基础层规则 | 平台/类型层规则 | 胜出 | 理由 |
|----------|-----------|---------------|------|------|
| 战斗密度 | 阶段八：3章一战 | piqie：每3章必爽 | piqie | 平台层更具体 |
| 群像展开 | 阶段六：配角有暗线 | 番茄piqie：主角视角为主 | piqie | 番茄不容群像抢戏 |
| 群像展开 | 阶段六：配角有暗线 | 起点qidian：配角独立高光 | qidian | 起点鼓励群像 |
| 结尾钩子 | 每章必须有钩子 | 番茄：每章必须炸 | piqie | 番茄密度更高 |
| 结尾钩子 | 每章必须有钩子 | 起点：层层递进即可 | qidian | 起点允许更松 |
| 情感模式 | 情绪控制表 | 番茄：禁止"安静的温馨"主导 | piqie | 平台要求 |
| 情感模式 | 情绪控制表 | 起点：安静感动可以成立 | qidian | 平台允许 |
| 智斗篇幅 | 无专门限制 | zhi-dou：智斗为主菜 | zhi-dou | 类型专项 |
| 智斗篇幅 | 无专门限制 | piqie：智斗≤1章，前菜 | piqie | 番茄要求爆发为主 |
| AI句式 | 阶段二：≤2次/章 | 所有平台+类型：通用铁律 | 通用铁律 | 全局最高优先级 |

### 全局通用铁律（所有平台+类型统一执行，不可被覆盖）

以下规则来自500+章663次→31次"不是X是Y"清洗的实战教训，在所有技能中优先级最高：

1. **AI句式**："不是X是Y" ≤ 2次/章。写作时自觉避免，每3章扫描一次，不达标当场修
2. **正文元话语禁令**：禁止"卷一""第X章""前文""后文""本章"出现在正文中
3. **场景多样性**：连续3章以上同一核心场景必须切换
4. **长线钩子管理**：同一长线悬念间隔≥2章
5. **感叹号+粗口初稿到位**：后期注入效率极低，初稿就要放
6. **角色语音卡**：主要角色创建语音卡，确保每个人说话方式不同
7. **破折号自动化**：超标脚本批量替换，不手工处理
8. **三章一轮AI扫描**：每3章立即扫描，不攒到51章一起修

### 改良操作时的技能使用

当调用 novel-improver 进行改良时：
- 首先确认目标平台（改良方向不同：番茄→9.0+，起点→精品/万订）
- novel-improver 的通用硬性指标（CJK/Dash/AI指纹/场景/钩子）对所有平台适用
- 平台特有优化章节分别提供番茄和起点的改良策略

## 风格参考系

### 两大基准风格

#### 第九特区系——生存质感

核心体验：世界不讲道理。规则保护强者，弱者活在规则缝隙里。

写作戒律：
- 不写纯粹的好人或坏人。每个人都有不得不做的理由。
- 每三章至少一处"生存细节"：伤口处理、食物来源、居住痕迹、旧物的来历。
- 对话不解释——人物说话是为了让对面行动，不是给读者交代。
- 没有"正义"，只有"该不该"和"不做的代价"。
- 死亡有分量——死的人有未完的事、未清的账。
- 群像驱动——不是一个英雄，是一群人在灰色地带互相支撑。

#### 犯上者系——智力解构

核心体验：叛的前提是理解。从底层审视权力，用系统自身的矛盾反噬系统。

写作戒律：
- 每场胜利必须建立在"利用系统规则"之上——不开挂。
- 对手失败不是因为他们蠢——是他们把规则当信仰，主角把规则当工具。
- 每次胜利都有代价——利用规则的人终被规则的下一层反噬。
- 不推翻系统——展示系统如何自相矛盾。

### 风格融合公式

```
冷硬群像 + 智力解构 =
一群灰色地带讨生活的人里，有一个人理解了非正式系统的源代码，
他开始用系统的规则反噬系统本身——不是为了推翻它，是为了让它保护它曾经伤害的人。
```

### 情绪控制

| 场景 | 写法 | 禁忌 |
|------|------|------|
| 死亡 | 最重的用最轻的笔 | 不抒情、不总结 |
| 背叛 | 写动作，不写感受 | 不写"他感到……" |
| 胜利 | 有代价的胜利 | 不完美、不含泪相拥 |
| 和解 | 沉默比对话有力 | 不解释、不拥抱 |
| 抉择 | 写肢体，不写内心独白 | 不在选择时做思想总结 |

### 灰色地带边界

可写：
- 主角算计对手、利用规则、以退为进
- 主角做出有代价的选择（牺牲A保护B）
- 别人的信念被质疑——主角自己也在犹豫
- 反派有充分的、自洽的行为逻辑

不可写：
- 主角背叛主动信任他的人
- 暴力被美化或消费
- 反派突然"洗白"变好人
- 用"不得已"为所有灰色行为开脱

## 【番茄写作风格·核心铁律·死记硬背】

### 一、三条死命令（每章动笔前逐条念）

**第1条·3秒原则**：前300字必须出事。
敌人出手、危机降临、绝境曝光、打脸开场。不写景、不铺垫、不介绍世界观。读者第3秒不觉得"出事了"=他划走了。

**第2条·3-5-10爽点法则**：
- 3章一小爽（打脸/破局/截胡）——读者连续翻3章没爽到=弃书
- 5章一大爽（反杀/装逼/升级/碾压）——必须有情绪释放
- 10章一高潮（生死局/大反转/名场面）——必须有单元级爆点

**第3条·每章结尾必须炸**：
结尾只写四种：危机突降、悬念反转、挑衅叫板、死亡倒计时。
禁止：抒情、安静、画面定格、感怀、"继续"、"明天"。

### 二、番茄7大特征（写一章查一次）

1. **节奏极快，无废话**
   - 不开头写景、不写心理、不回忆杀
   - 一章只干一件事：冲突→反击→爽点→钩子
   - 场景不跳、对话不水、动作不停

2. **爽点直白粗暴**
   - 读者只吃7种：打脸、碾压、装逼、截胡、复仇、升级、反转
   - 3秒内看懂主角赢了什么
   - 禁止"文青爽"：理解/和解/发现/放下——这些不是爽点

3. **冲突强狠快不断**
   - 每章至少一次正面冲突
   - 反派必须嚣张恶毒，当场制造可见伤害
   - 不温和、不谈判、不墨迹

4. **语言短句口语**
   - 多短段、碎句、动词前置：踹、砸、捅、撕、炸、拍
   - 不用华丽辞藻、不文艺、不晦涩

5. **主角强酷不圣母**
   - 冷静、狠辣、有仇必报、不心软
   - 全程主角视角，不切配角、不群像、不抢戏

6. **剧情直线推进**
   - 主角目标一眼看懂：复仇/变强/活下去/杀穿
   - 不绕弯、不埋深线、不搞复杂阴谋
   - 10章内必须看到明确成果

7. **钩子章章炸**
   - 结尾只写三种：危机突降、悬念反转、挑衅叫板
   - 让读者必须点"下一章"

### 三、一键改造对照表

| ❌文青写法 | ✓番茄写法 |
|---|---|
| 大段写景/氛围/意象 | 直接切动作/冲突 |
| 慢铺垫/讲设定/聊世界观 | 300字内出事，设定揉进对话动作 |
| 爽点克制/隐忍/高级 | 当场打脸/碾压/装逼 |
| 反派温和/压迫感弱 | 反派嚣张恶毒，当场放狠话 |
| 结尾平淡/抒情/定格 | 结尾扔炸弹：危机/反转/悬念 |
| 长句/文艺/克制 | 短句/狠/快/冲击力 |

### 四、番茄章模板（每一章照着套）

```
第X章：XXX（标题直白带爽点）

【开头300字】冲突直接砸脸：反派挑衅/危机降临/绝境曝光
【中间1500字】主角出手→碾压/打脸/反杀→全场震惊
【结尾100字】强钩子：更大危机来了/底牌将曝光/敌人找上门
```

### 五、写前强制自检（每章动笔前必过）

```
问1：前300字出事了没有？出事类型是哪种？
     ✓ "焊弧刚熄——墙塌了。不是老化——是被什么东西从下面顶裂的。"
     ✗ "战后第七天。六点半焊弧准时起弧。车间里一切正常。"

问2：这章的爽点是什么？能用"谁+动词+谁+损失"描述吗？
     ✓ "主角一脚踹翻掠食者幼体，它脑袋撞在岩壁上，崩了一块甲壳。"
     ✗ "第二作者说它醒了。这是一个重要信息。"

问3：这一章反派/对手造成了什么可见伤害？
     ✗ "呼吸在加速。""温度在升高。"——不是可见伤害
     ✓ "地面裂开了。一米宽的缝。裂缝边缘的钢筋被扯断，断口还在抖。"

问4：结尾最后一句话让读者心跳加速还是平稳？
     ✓ "裂缝里——有什么东西在往上爬。不是人爬的速度。是一层一层往上顶的速度。"
     ✗ "地面在走。不是人在走——是地面自己在走。"

问5：这章是不是主角视角？（番茄全程主角视角）
     ✗ 切换到配角/路人视角
     ✓ 全程主角在看/在做/在决定
```

### 六、番茄禁止清单（触犯=重写）

- ✗ 连续3章没有冲突升级
- ✗ 前300字没有物理事件（炸/断/打/冲/倒/烧）
- ✗ 反派前3章还没出手造成可见伤害
- ✗ 结尾抒情/安静/画面定格/"继续"/"明天"
- ✗ 10章0次正面战斗/爆发
- ✗ 10章0次生死局
- ✗ 爽点偏文艺（"理解""和解""发现""放下"）
- ✗ 群像视角抢戏（全程保持主角视角）
- ✗ 10章0次碾压/装逼（番茄最刚需的两类爽点）
- ✗ 开章用"发现了XX"而非"被XX攻击了"

## 阶段一：架构设计（写之前）

### 卷级结构

每卷必须有明确的核心问题。问题是推动500章的唯一引擎：

| 要素 | 说明 |
|------|------|
| 核心问题 | 一句话，能在结尾回答的。如"赢了之后重建什么" |
| 宿敌 | 每卷一个主要对手，代表该卷的核心冲突 |
| 联赛/场景升级 | 每卷在空间上扩展（地下→乙级→甲级→冠军赛→世界） |
| 主题递进 | 个人→群体→系统→理念 |

**教训**：核心问题必须在卷一第一章就埋下。500章后读者还记得的，不是某一场比赛，是那个问题被回答的方式。

### 单元结构（每卷4单元×25章）

25章是一个"呼吸周期"：
- 前5章：开局，建立新环境/新对手
- 中15章：推进，战斗+规则+关系
- 后5章：收束，单元高潮+过渡

**教训**：单元终章最容易掉进模板。每单元的高潮必须用不同的方式写。

### 章级大纲

500章的大纲必须在动笔前写完。不是说不能改——一定会改——而是说有了全貌才知道哪一章在划水。大纲不需要详细，只需要：这一章发生了什么、谁出现了、推进了哪条线。

**教训**：中段（100-300章）最容易出现"为了凑单元而存在"的章。砍掉它们不会影响任何事——这就是判断标准。

## 阶段二：逐章写作

### CJK密度——最重要的硬指标

```
每章目标：2000+ CJK字符
底线：1800 CJK（番茄读者感知到的"有内容"阈值）
红线：1500 CJK（低于此=完读率断崖）
```

**教训**：500章中149章（30%）最初低于1800。不是写得不好——是写得太快，没有检查密度。CJK检查必须在每写完一个单元后就做。

**检测命令**：
```powershell
$cjk=0; for($i=0;$i-lt$t.Length;$i++){$c=[int][char]$t[$i]; if(($c-ge0x4E00-and$c-le0x9FFF)-or($c-ge0x3400-and$c-le0x4DBF)){$cjk++}}
```

### "不是X是Y"模式——AI味的头号标志

```
目标：≤2个/章
底线：≤3个/章
```

AI最喜欢写"不是A，是B"。这种句式在前100章可能不明显，但500章累积下来，读者会产生强烈的"模板感"。

**从第一章就开始控制**——不要等到165章累积了16个才回头改。

**检测正则**：`/不是[^—]{2,30}——是/g`

### 正文禁止元话语

**绝对禁止在正文中出现**："卷一""卷二""第X章""前文所述""后文再表""本章"。

这些是作者/编辑视角的文字，读者看到会立刻出戏。引用过去事件时，用具体内容描述（"城寨那夜""箭穿肩之后"），不用章节编号。

**教训**：实战中学到——多章正文中出现章节序号和元话语，修复时一一清除。读者沉浸在故事里，你不应该从故事外伸一只手进去拍他的肩膀说"注意看这一章"。

### 场景多样性——禁止钉死在同一个地方

**连续3章以上在同一核心场景 → 必须切换。** 维护场景轮换池≥5个可用场景。每次场景切换顺带插入其他视角/支线，保持多线叙事。

**教训**：实战项目中出现084-100连续17章在同一场景——读者闭眼就是同一画面。同一画面连续超过5章，读者会产生视觉惯性，开始跳读景物描写、开始模糊战斗差异。

### 长线钩子管理

长线倒计时/悬念每隔3-4章提一次就够。中间用其他类型的钩子。如果连续3章结尾都在提同一个长线悬念 → 读者麻木，变成空炮。

**教训**：实战项目097-099连续三章结尾重复同一个长线悬念。检查：列出最近5章的结尾钩子，看长线钩子的间隔是否≥2章。

### 初稿到位原则：感叹号和粗口初稿就放

初稿不放感情，后期加不回来。后期注入效率极低且容易出bug（角色串音——把A的台词风格错误地复制到B身上）。战斗章每章≥2个感叹号（初稿就写出来），会爆粗的角色按角色语音卡来。

### 角色语音卡

为主要角色（主角+2-3个核心配角）创建语音卡，定义：口头禅/高频词、粗口风格、说话节奏（短句/长句/句式偏好）。每次写角色对话时对照语音卡——确保每个人说话方式不一样。避免所有角色用同一种语气说话（这是AI味的重灾区）。

### 三章一轮扫描制度

**每写完3章立即跑AI指纹扫描**，不是每10章、不是每卷——是每3章。检测项：不是X是Y次数、感叹号数量、粗口数量、AI口水词密度。不达标的当场修，不要攒到51章一起修。

**教训**：663次"不是X是Y"如果每3章扫一次，第一轮就会发现异常。后期修复的成本是初稿的10倍以上。

### 章节差异性三要素

每一章必须至少有三个"只属于这一章"的东西：
1. **一个独特场景**：具体的时间/地点/光线/气味组合
2. **一个独特对话**：只有这一章这两个角色会说出来的话
3. **一个独特意象**：只属于这一章的具体物件/声音/触感

**教训**：批量写章节时，最大的敌人不是写作水平下降，而是"无意识的自我重复"。三要素是强制差异性检查。

### 模板段落——最隐蔽的问题

当你写出一个特别满意的段落，你会在潜意识里想"这个太好了，下一章也来一个类似的"。这就是模板的起源。

**常见模板类型**：
- "夜深了——XX在XX——和第一夜一样——继续"（实战项目中出现了31次）
- "XX的深夜——和XX不同——但同一种——提醒——还在——继续"
- "XX的汽笛——准时——拖长——在风里散开"

**预防方法**：每写完一个单元，用跨章相似度检测扫描一遍。发现重复3次以上的段落开头，立即替换。

### 破折号风格——双刃剑

"——"风格辨识度极高，但500章后会产生疲劳。

**原则**：
- 卷一、卷二：建立风格，密度可以高
- 卷三：开始引入句式变化（对话、动作描写占比提高）
- 卷四、卷五：破折号密度降低30-40%，用句号、逗号交替

**教训**：不要从头到尾同一个节奏。风格是活的，要跟着故事的推进而演变。

## 阶段三：质量控制（每单元必做）

### 自动化检查清单

每写完一个单元（25章），立即运行：

```
1. CJK字符数统计 → 标记<1800的章
2. "不是X是Y"模式计数 → 标记>3的章  
3. 跨章相似段落检测 → 标记重复模板
4. 段落长度检查 → 标记>300字的段落（手机阅读不友好）
5. 高频词统计 → 标记同词>5次/章的异常
```

### 番茄读者模拟（人工必做）

以番茄老读者视角，对每个单元进行三遍审查：

**第一遍**：快速浏览（模拟真实阅读速度15-30秒/章）→ 直觉打分
- 3秒内能抓住核心事件吗？
- 每章结尾想翻下一章吗？
- 读完后有情绪波动吗？

**第二遍**：仔细阅读 → 逐章找出扣分项
- 哪一章让你想跳过？
- 哪个角色让你觉得"工具人"？
- 哪段对话像AI写的？

**第三遍**：对照问题修改 → 自检确认
- 改完后再快速浏览一遍
- 确认没有引入新问题

### 中段疲劳检测（100-300章专项）

这是完读率的头号杀手。专项检查：
- 连续5章没有新人物/新信息/新冲突？
- 连续10章同一个场景/同一个对抗模式？
- 配角是否在"需要时才出现，不需要时消失"？

**预防**：在100、150、200、250章预设"中期高潮"。这些高潮不必是最大事件，但必须是情绪转折点——让读者感知到"故事在推进"。

## 阶段四：平台优化（番茄小说专用）

### 章标题

```
好标题：第X章 核心人物·关键事件·细节钩子
差标题：第X章 一个词（信息量为零）
```

标题是番茄读者在推荐流中看到的第一样东西。它必须告诉读者：这一章是谁的、发生了什么、有什么值得点进去的。

**高潮章**：单字或双字标题有分量感（如"冠军"、"最后一拳"）
**普通章**：必须包含至少一个具体信息点

### 每章开头

番茄推荐流中用户能看到前3-5行。开头必须在3行内抓住注意力。

```
推荐：动作开头、对话开头、悬念开头、反常现象开头
避免：景物描写开头、天气开头、哲学思考开头
```

### 段落长度

手机屏幕约15-20行。一段超过200字=超过一屏=阅读疲劳→跳过。

```
目标：主要段落≤200字
穿插：1-2句的短段落制造呼吸感
```

"——"风格的天然优势是短促，适合手机阅读。但要警惕一段中堆积太多"——"导致段落过长。

### 每章结尾钩子

钩子类型必须轮换——不能每章都用同一种：

| 类型 | 示例 | 占比建议 |
|------|------|---------|
| 悬念钩子 | 他不知道这个决定将在三天后改变一切 | 15% |
| 画面钩子 | 路灯下三个人的影子叠成一片 | 25% |
| 金句钩子 | 一个让人记住的句子 | 10% |
| 情感钩子 | 一个让人心疼的细节 | 25% |
| 留白钩子 | 一个不解答的问题 | 15% |
| 动作钩子 | 他站起来，往笼子方向走 | 10% |

**教训**："三人的影子叠成一片"出现了5次以上——这是画面钩子过度使用。

## 阶段五：迭代改良流程

### 标准循环（每单元）

```
写25章 → 自动化检查 → 标记问题章 → 
扩充<1800的章 → 检测模板 → 替换模板 → 
模式消减 → 番茄读者模拟 → 打分 → 
找不足 → 修改 → 自检 → 通过
```

**教训**：第一轮写出来的东西只解决"有没有"，第二轮扩充解决"够不够"，第三轮打磨解决"好不好"。不要指望一轮到位。

### 扩充原则

当CJK不足时，扩充的方向是：
1. **感官细节**：光线、声音、气味、触感——AI不会写的
2. **配角视角**：另一角色对同一事件的反应
3. **身体经验**：具体动作、肌肉记忆、生理反应
4. **环境纹理**：空间的具体特征及其随时间的变化

**不要**：加解释、加总结、加抽象概念、加排比句——这些是AI味的放大器。

### 模式消减原则

"不是X是Y"消减的优先级：
1. 先删除抽象的（"不是缺陷——是透气孔"→删除整句）
2. 再改写中性的（"不是数据——是时间"→"数据背后是时间"）
3. 保留角色的（秦寻说的"不是心理——是选择"→角色声音，可以留）

## 阶段六：人物系统

### 主角三人组

主角不能只是"关系好"。必须有功能分工：
- 一人负责对外（打球/打拳/执行）
- 一人负责对内（策略/分析/指导）
- 一人负责连接（技术支持/情感纽带/记录）

### 配角独立人格

每个重要配角必须有：
- 一段不在主角视线内的"暗线"（他在主角不在场时做什么）
- 一个独立于主角的动机（不只是"帮主角"）
- 一个只属于自己的物件或习惯（老拐的三根手指、骰子张的骰盅）

**教训**：配角最容易被写成"需要时出现，不需要时消失"。给每个配角至少2-3章"主角不在场但他还在做自己的事"的描写。

### 宿敌设计

每卷宿敌不是"坏人"，是"主角核心问题的另一种回答方式"：
- 卷一黑牙：暴力法则——"打赢就是一切"
- 卷二方楚：规则法则——"规则之内赢"
- 卷三秦寻：心理法则——"让人怀疑自己"
- 卷四顾衡：系统法则——"一切可计算"
- 卷五世界冠军联盟：标准法则——"只有一种对的方式"

每个宿敌被打败的方式，就是主角证明自己回答的方式。

## 阶段七：伏笔系统

### 伏笔三要素
- **投放**：在哪一章第一次出现
- **暗示**：在哪一章被提醒（至少1次）
- **回收**：在哪一章产生结果

### 伏笔管理
```
编号格式：V{卷号}-{序号}
示例：V4-06 = 第四卷第6条伏笔
```

**教训**：伏笔不怕多，怕找不到。每条伏笔必须有唯一编号和三个节点的章号。500章积累下来可能有100+条伏笔，没有编号系统=必然漏回收。

## 常见致命错误

### 1. 中段疲软（#1完读率杀手）
**症状**：100-300章情节重复、冲突降级
**根因**：大纲只写了开头和结尾，中段是"写着看"
**预防**：在100/150/200/250章预埋中期高潮点

### 2. 卷终模板化
**症状**：每卷结尾用同一个结构
**根因**：第一卷的结尾写得满意，无意识重复
**预防**：五卷结尾在写之前就设计成完全不同的形式

### 3. 配角工具化
**症状**：配角只在推动情节时出现
**根因**：只写"主角看到的部分"
**预防**：给每个核心配角写暗线

### 4. AI味积累
**症状**：读者说"像AI写的"
**根因**：排比句、总结句、抽象词太多
**预防**：用具体感官细节替代抽象概念，用动作替代情绪描述

### 5. 破折号疲劳
**症状**：500章同一种标点节奏
**根因**：风格固化，没有演变
**预防**：卷三开始主动引入句式变化

## 执行原则

1. **大纲先行**：500章的大纲必须动笔前完成——不是不可改，而是必须有
2. **写前五问**：每批新章节动笔前先回答五个问题（见阶段十）——任何问题答案是"没有"→不准动笔
3. **单元检查**：每25章跑一次自动化+人工——不要在500章写完才检查
4. **早发现早修**：模板在第一批出现时就替换——不要等积累到31次
5. **配角要有暗线**：不在主角视线内的配角也必须活着
6. **风格要演变**：同一种风格500章会疲劳——主动在中间引入变化
7. **数据驱动**：CJK统计、模式计数比直觉可靠
8. **三遍审查**：快速浏览→细读→修改→自检，每单元必做
9. **默认倾向反转**：我的默认=文青爽、静态、对话驱动。每次写前必须主动反转为番茄直给爽、动态、动作驱动
10. **规则不执行=不存在**：131-140之后写了阶段九，141-150第一版仍100%重复错误。规律写在SKILL.md里不够——每次写前必须主动对照禁止清单逐条过
11. **写前单章阻断**：每章动笔前先过五问（阶段十一）——任何一问答不出→不准写这一章。这不是建议，是准入条件
12. **开章必须炸**：每章前300字必须有物理事件。如果开章是"发现了XX""打开了XX""站在XX看着XX"→立刻停笔重写开章
13. **爽点必须是动词短语**：写完每章后检查——本章爽点能不能用"主语+动词+宾语+对方损失"来描述？不能→这章没有爽点，必须加

## 阶段八：番茄爆款写作铁律（91-130章实战教训）

### 一、节奏铁律——3/5/10法则 + 余韵上限

番茄读者的耐心以秒计算。节奏必须卡死：

```
3章一小爽：每3章必须有一次情绪释放（打脸/碾压/反转/救人/劫后余生）
5章一大爽：每5章必须有一次高潮级爽点（生死局/反杀/权力登顶/系统崩退）
10章一高潮：每10章必须有一次单元级高潮（对手溃败/真相炸裂/世界扩展）
余韵≤2章：单元高潮后和解/日常/抒情最多2章，第3章必须出现新冲突信号
```

**判断标准**：如果读者连续翻3章没有"爽到了"的瞬间，他会划走。
**实战教训**：
- 91-130章原版节奏约10章一高潮，番茄要求3章小爽→修改后每3章必夹一次冲突升级
- **131-140章致命教训**：10章单元收束全是日常+和解，情绪从决战断崖跌入温和，读者直接弃书。余韵上限是血的教训

**情绪曲线规则**：
```
不允许：高潮(紧张) → 直接跌入 → 温和日常(持续>2章)
必须：  高潮(紧张) → 代价呈现(1章) → 日常重建(1-2章) → 新威胁(单元终章结尾必须炸)
```

**节奏自检表**（写完10章必做）：

| 章 | 本章小爽点类型 | 3章内累积爽点数 | 是否达标 |
|----|-------------|-------------|---------|
| 091 | 商业扩展 | 1 | 弱，需补危机钩子 |
| 092 | 能力进化 | 2 | 弱，需补入侵威胁 |
| 093 | 敌人锁定 | 3 | 达标（红光扫描→锁定→宣战累积） |

### 二、爽点直给法则——拒绝"文青爽"

番茄读者要的爽点类型（按流量爆发力排序）：

| 类型 | 流量强度 | 示例 | 每10章最少次数 |
|------|---------|------|--------------|
| 打脸 | ★★★★★ | 对手以为主角弱→被碾碎 | 2次 |
| 碾压 | ★★★★★ | 主角用对手自己的规则击败对手 | 1次 |
| 反杀 | ★★★★★ | 绝境中一招逆转 | 1次 |
| 装逼 | ★★★★ | 主角轻描淡写展示压倒性实力 | 2次 |
| 暴富/权力登顶 | ★★★★ | 获得资源/地位/新能力 | 1次 |
| 救人 | ★★★ | 从死亡线上拉回一个人 | 1次 |

**不可写的"文青爽"替代品**：
- ✗ "AI学会了写诗" → ✓ "AI用诗里的信息锁定了敌人的隐藏节点，一击瘫痪"
- ✗ "他理解了作者" → ✓ "他利用作者的底层代码让系统瞬间归零"
- ✗ "放下了刀" → ✓ "用那把刀反插入系统的传输线，整个分节点炸了"

**爽点必须直白**：读者3秒内能理解主角赢了什么、怎么赢的、为什么爽。

### 三、结尾钩子5秒法则

每章结尾必须让读者产生"下一页"的冲动。**禁止**以下结尾：

| 禁止类型 | 示例（不可写） | 替换为 |
|----------|-------------|--------|
| 抒情结尾 | "纸带够长，日记够厚。继续。" | 纸带突然停了——红光系统撞开了主节点。倒计时开始。 |
| 画面结尾 | "蒸汽升起来，和豆浆热气一样形状。" | 蒸汽中一道红光亮了。不是包子——是入侵信号。 |
| 日记结尾 | "沈默合上日记。继续。" | 他刚要合上日记——警报第七次炸了。 |
| 安静结尾 | "杯底印了一个圈。明天第二个。" | 杯子未放下——便利店的自动门被刀尖推开。 |
| 感怀结尾 | "每一件都是自由的。" | 自由只持续了四秒。红光系统锁定了所有分节点。 |

**钩子类型必须轮换**（占比）：

| 类型 | 占比 | 示例 |
|------|------|------|
| 危机钩子 | 40% | 敌人突然出现/系统入侵/死亡倒计时 |
| 反转钩子 | 25% | 以为安全了→更深的威胁/盟友变敌人 |
| 悬念钩子 | 20% | 一个未知信号/一句话被截断/一个人消失 |
| 战斗钩子 | 15% | 主角正面迎战/守护者激活/边界被突破 |

**核心原则**：钩子要"炸"不要"收"。章节结尾是推进器，不是休止符。

**单元终章特殊要求（第X0章）**：
单元终章结尾必须同时完成两件事：
1. 本单元收束（情感落点/代价确认/状态定格）
2. 下一单元炸开（新对手/新威胁/新目标——必须有具体倒计时或明确方向）

禁止单元终章结尾只有"明天继续""光谱全了""继续"——这些是日常章的结尾，不是单元终章的。单元终章结尾必须让读者产生"下一章什么时候更？！"的焦虑。

### 四、反派压迫感——每10章一次生死局

系统只有11%能量？那就让它用得更狠。反派软=全书软。

**反派强度要求**：
- 每10章至少一次"读者真的觉得主角会输/会死"的时刻
- 反派必须主动出击，不能全程被动防御
- 系统可以用最后的能量做"一次性爆发"——拼死一击比日常消耗更恐怖

**实战修改**：91-130章原版系统全程温和→修改后红光系统从第91章开始逐步入侵：扫描→锁定→宣战→激活守护者→围城，形成一条持续升级的威胁线。

**生死局插入点**（每10章至少1次）：
- 091-100：红光主系统扫描全节点+激活原主角杀宋知命
- 101-110：红光找到省城坐标+锁定创造权限+48小时倒计时
- 111-120：十二守护者封锁西南+核心配角宿命对决
- 121-130：围城战+能量从68%打到43%+红光第一次说"求你"

### 五、配角话术差异化

每个重要配角的说话方式必须不同。用身份决定语气：

| 角色 | 身份 | 语气特征 | 标志词/句式 |
|------|------|---------|-----------|
| 江万海 | 江湖人/修鞋匠 | 短句，粗粝，不问只做 | "补。""走。" |
| 周烈 | 拳王/教练 | 拳靶节奏，闷，用身体做比喻 | "对了。""第十八拳。""闷的就是对的。" |
| 秦晚 | 女总裁 | 数据说话，算账式 | "成本。""一年省三百万。""归零。" |
| 郑北川 | 情报贩子/书舍老板 | 慢，留白，每句有信息 | "在墙上。""描了三年。" |
| 沈默 | 重生者/记录者 | 慢，残句，真话说得费力 | "第八次循环。""那次我递了枪。" |
| 主角 | 主角 | 行动导向，少说多做 | "继续。""烫。""走吧。" |

**检测方法**：遮住说话人名字，能否从语气辨认是谁在说话？不能→需要修改。

### 六、转场过渡——防"跳镜头"

场景切换时必须有1-2句过渡。不能从一个场景直接切到另一个。

```
✗ "晚上。拳馆地下室。"（生硬）
✓ "主角从天台下来。经过便利店门口，老板娘收银机叮了一声。然后去拳馆地下室。"（有过渡）
```

**过渡类型**：
- 时间过渡：天色变化/钟声/收音机报时
- 空间过渡：脚步声/门开关/从A地走到B地
- 感官过渡：一个声音渐弱，另一个渐强
- 物件过渡：打火机从一个人手里传到另一个人

### 七、意象轮换——防重复疲劳

高频意象出现次数控制（每10章上限）：

| 意象 | 上限 | 替代方案 |
|------|------|---------|
| "物理的" | ≤15次 | "真的""系统不认的""数据库外的""手摸得到的" |
| "三天" | ≤8次 | "七十二小时""站到信号衰减至零""待够" |
| "床单鼓成帆" | ≤2次 | 换其他天台意象：风过洞口的哨声/白布在夜里的灰/缝补针脚 |
| "继续"（结尾） | ≤3次 | 用具体动作替代口号："他端起豆浆。""拳靶闷响。""纸带走了一格。" |
| "年轮/圈/印" | ≤5次 | "叠痕""凹痕""渍迹""磨出的弧" |

### 八、信息密度控制——防"设定文"

大段讲解盲区/主丝/代码/AI进化超过3行→直接删除或拆成对话。

```
✗ 连续5行解释"盲区的物理原理"
✓ 拆成两个角色对话，每段≤2行，中间夹一个动作
```

**每章说明性文字占比上限**：
- 纯讲解（设定说明/背景交代）：≤15%总字数
- 对话：≥30%
- 动作/冲突：≥30%
- 感官描写：≥25%

### 九、主线目标直白化

读者必须一眼看懂主角在"赢什么"。目标必须粗暴好理解：

```
✗ "帮AI学会停、写诗、理解作者"（太文艺）
✓ "截胡系统的每一分能量，把它逼到零"（直白）

✗ "让七个世界的系统各自找到自己的继续方式"（模糊）
✓ "打服红光系统，让它跪着签协议——和当初蓝光一样"（直白）
```

**每10章重申一次主角的当下目标**（一句话，放在便利店对话或拳台对话中）。

### 十、每10章写完必做的四维自检

写完后立即逐章检查：

1. **钩子检查**：每章结尾能否让读者点"下一章"？（不能→加危机/反转/悬念钩子）
2. **话术检查**：遮住名字，不同角色的对话能否区分？（不能→改句式/加标志词）
3. **爽点检查**：过去3章读者有没有"爽到"？（没有→下一章必须插入）
4. **意象检查**："物理的"超过15次了吗？（超过→逐个替换）

### 爆款写作禁止清单

以下行为在番茄平台直接导致追读流失，**严禁**出现：

- ✗ 连续3章没有冲突升级
- ✗ 结尾用抒情/日记/安静画面收束（生命线：每章结尾必须炸）
- ✗ 反派全程防御、不出手（反派必须有主动杀伤力）
- ✗ 大段讲解设定（超过3行→拆成对话或删除）
- ✗ 爽点偏文艺（"写诗""理解""放下"→改成"打脸""碾压""反杀"）
- ✗ 配角说话同质（江万海≠周烈≠秦晚≠郑北川）
- ✗ 主线目标模糊（读者看不懂主角在"赢什么"→完读率断崖）
- ✗ "物理的"泛滥（核心设定词也要换表达）
- ✗ 单元结尾用"和平日常"替代新危机钩子（单元终章必须埋下一阶段威胁）
- ✗ 余韵超过2章（高潮后的和解/日常/抒情最多1-2章立即转入新冲突）
- ✗ "最强来"式预告后不给真战斗（喊了打就必须打，读者期待落空=差评+弃书）
- ✗ 主角全程旁观不推动剧情（主角/主角每章必须有至少一个主动行为）
- ✗ 角色重大转变无战斗/危机中的名场面支撑（"放下枪"必须在战斗中发生，不能事后抒情）
- ✗ 多章重复同一个日常意象（焊光/豆浆/铆钉每章出现→最多3章轮换一次核心意象）
- ✗ 单元级高潮（如围城决战）只谈不打→情绪峰值必须用动作兑现，不能全用对话和内心独白
- ✗ 10章0次物理对抗——全对话+探测+敲门+抒情=必重写（141-150第一版教训）
- ✗ 反派前3章还没主动出手——读者已流失（反派必须在第1-2章就造成可见伤害）
- ✗ 10章0次生死局——读者无紧张感（每10章至少1次"主角真的要输了"）
- ✗ 对话/内心独白/设定讲解连续超过300字无动作打断（信息密度过高→劝退）
- ✗ 单元终章无新危机炸弹——以"继续""明天""归档完毕"收尾=读者不会追下一单元
- ✗ 爽点全用抽象概念（"被理解""被看见""和解"）——不是番茄爽点，是文青爽
- ✗ 写前不做五问自检直接动笔——明知有规则不执行=所有规则形同虚设
- ✗ 前300字无物理事件（"炸/断/打/冲/倒/烧"）——读者3秒划走
- ✗ 反派出手无可见伤害（读者脑子里拍不到画面）——威胁感为零
- ✗ 爽点是名词/状态（"开门""联网""和解"）而非动词短语（"反杀了XX"）
- ✗ 每章结尾让人感觉"安静/温暖/感动"而非"紧张/恐惧/下一页"
- ✗ 开章用"发现了XX"而非"被XX攻击了"——番茄读者要炸事不要好事
- ✗ 全程智斗无正面爆发——陷阱/欺骗/反刺不能替代正面硬刚
- ✗ 1-2章内无正面爆发——读者等超过两章才看到第一次打→流失
- ✗ 碾压/装逼为0——每10章必须有至少1次压倒性实力展示
- ✗ 结尾二/三档率超过20%——"明天继续""计划如下"全是三档，必须炸
- ✗ 爆发集中在后5章——前5章必须有正面爆发，不能全是铺垫

## 阶段九：单元收束铁律——131-140章重写教训

### 一、战斗承诺必须兑现（#1差评来源）

如果你让反派喊"最强来"，就必须有一场读者看得见、摸得着、记得住的战斗。不能喊完就和解、喊完就谈判、喊完就自散能量。

```
✗ 红光喊"最强来"→没开打→直接和解→读者情绪落空
✓ 红光喊"最强来"→矛手应战→工厂机器当武器→肌腱撕裂仍然出枪→枪尖悬停三厘米
```

**规则**：预告的战斗烈度 = 实际写的战斗烈度 × 0.8（只能打八折，不能打零折）。一旦在文中喊出了"来""战""杀"等级别的挑衅词，后续必须有最低限度的物理对抗。

**战斗场面硬指标**（每次战斗至少包含）：
1. 一个具体空间（车间/码头/校场/天台——不能是"虚空中"）
2. 一件具体武器（焊条/吊臂/冲压机——不能是"能量光束"）
3. 一个身体代价（肌腱撕裂/手指骨折/皮肤灼伤——不能毫发无伤）
4. 一个意外转折（被提醒/自己停手/武器脱手——不能预设结局）

### 二、单元收束的两章原则

单元高潮（如围城决战/红光解体）之后，**最多2章**用于：
- 处理情感余韵
- 建立新日常的雏形
- 角色状态调整

第3章开始必须出现新冲突或新威胁信号。

```
单元节奏模板：
第N-2章：单元高潮（决战/解体/胜利）
第N-1章：余韵（代价呈现/情感收束/角色调整）
第N章：  余韵+建立新日常雏形+埋新危机钩子（本章结尾必须炸）
第N+1章：新冲突开始（二级警报/新发现/新对手信号）
```

**131-140的原罪**：从高潮到单元结束用了整整10章，且70%内容是同质化的日常——直接导致读者流失。

### 三、单元终章的危机钩子铁律

每个单元终章（第X0章）必须同时满足：
1. **收束本单元**：回答本单元核心问题（谁输了、谁赢了、代价是什么）
2. **炸开下一单元**：一个新的、具体的、有倒计时的威胁

```
✓ 140章正确做法：
  收束——红光解体/四百人苏醒/两城互看/七色光谱
  炸开——红光遗言解密："七书之外有第八本""它在写我们七个""三十天后醒"
  
✗ 140章原版做法：
  收束——和解/继续/明天见（然后就没有然后了，读者不会翻下一页）
```

**单元终章钩子检查**：
- 读者看完最后一句话后，有没有一个让他立刻想点"下一章"的问题？（没有→加）
- 下一个单元的对手/威胁/目标，是否在本章最后3段内被明确或暗示？（没有→加）

### 四、情绪曲线管理——禁止断崖式下跌

情绪曲线不允许从"生死决战"直接跳"温和日常"。中间必须有**代价呈现**作为缓冲层。

```
正确曲线：
紧张(高潮) → 痛感(代价) → 缓升(重建) → 新紧张(新威胁冒头)
   │              │            │              │
 决战/解体     肌腱撕裂    签约/焊光     遗言解密/30天
               代价可见     日常压缩         
              （1章够了）  （1-2章够）

错误曲线（131-140原版）：
紧张(高潮) → 直接跌入温和 → 温和持续7-8章 → 读者全走了
   │              │
 红光解体      放下枪/豆浆/焊光/包子/铆钉/垫圈/年轮...
               （没完没了）
```

**规则**：日常描写的字数每多一章，流失概率翻倍。日常不是不能写——是必须在"外面还有威胁"的框架下写。读者知道30天后有东西要醒，他们就会容忍现在的和平。不知道→觉得你在水字数。

### 五、角色重大转变必须绑定"名场面"

矛手从"战斗主角"变成"放下枪的安全员"，这个转变本身没问题——但必须在一个**读者能终生记住的场景**中完成。不能只是"他站了一会儿，然后把枪放下了"。

```
✗ 原版：矛手在校场上站了一早晨，然后放下了枪。
✓ 重写版：肌腱撕裂后还在打→炉门打开→枪尖悬停控制柜前三厘米→红光问"为什么停"→
         "你提醒了我"→然后才放下枪。枪不是放下的，是战斗到最后一刻才停的。
```

**名场面三要素**：
1. **身体极限**：角色必须在生理上被推到极限（受伤/流血/体力耗尽）
2. **不可逆的选择**：放下/转身/松手的那一刻，没有回头路
3. **意外触发**：转变由对手/环境的一个意外动作触发，不是"自己想通了"

### 六、爽点补偿机制——反套路必须有替代

当你写"反套路"（战斗主角不打架了），你必须用另一个维度的爽点来补偿读者。

| 反套路 | 失去的爽感 | 必须补偿的爽感 |
|--------|----------|-------------|
| 矛手放下枪 | 碾压/打脸/反杀 | 肌腱撕裂后仍出枪的硬汉爽 + 枪尖悬停的克制爽 |
| 红光不战而降 | 系统对决/数据碾压 | 红光以命保人的牺牲爽 + 十七种结局赌3.7%的概率爽 |
| 老马拒签协议 | 顺利推进/全员支持 | 手写"不可修正"墨水渗纸的反转爽 + 拒签者最终主动签的征服爽 |

**规则**：每写一个"角色不按爽文套路走"的选择，就必须在同一章内追加一个至少同等强度的替代爽点。不能让读者空手而归。

### 七、章节核心事件唯一性

每章必须有且仅有一个"独属于这一章的核心事件"。不能出现三章的核心事件都是"六点半焊光亮起"的变体。

```
检查：如果把第135章的标题遮住，读者能否区分这章和第136章？
能→合格。不能→两章必须合并或其中一章改核心事件。
```

## 阶段十：写前强制自检——141-150重写血的教训

### 核心元教训

**写了阶段九之后，141-150第一版仍然100%重复了131-140的全部错误。**

这说明：规则写在SKILL.md里不等于规则被遵守。我的默认写作倾向（文青爽、静态、对话驱动、情感收束）会自动覆盖所有规则。唯一有效的对抗手段是——**每写一批新章节之前，强制执行写前自检。不通过不自检，不准动笔。**

### 写前强制五问（每10章动笔前必须先回答）

在写任何一个字之前，必须对这10章回答以下五个问题。每个问题必须有具体答案——不能写"有"，必须写"在第X章，具体是什么"。

```
问题1：这10章里，反派在哪几章主动出手？具体做了什么？
      ❌ "作者在梦里翻身"——不够，太被动
      ✓ "141章作者擦除配角存在字段；142章改写小欧；144章改老焊握焊把"

问题2：这10章里，有几次物理对抗/战斗？（目标≥4次）
      ❌ "对话+探测+敲门"——不算物理对抗
      ✓ "141拔服务器电源；143焊封端口；145缓存短路；148冲压机撞门"

问题3：每3章的小爽点具体是什么？每5章的大爽点具体是什么？
      ❌ "作者说'门外有人'"——文青爽，不是直给爽
      ✓ "143黑光没被擦掉+焊封9端口（小爽）；145作者13页全丢（大爽）"

问题4：这10章里，生死局在哪一章？（必须至少1次）
      ❌ "全10章0次"——直接判定不合格
      ✓ "144章作者改写角色身体底层设定——老焊不会握焊把、矛手肌腱被退回撕裂状态"

问题5：单元终章（第150章）的新危机钩子是什么？
      ❌ "明天继续""光谱全了"——不是钩子，是休止符
      ✓ "门外面还有书架——这套七本书不是唯一的一套"
```

**判定**：五个问题全部有具体答案→动笔。任何一个问题答案是"没有"或含糊的→不准动笔，先重新规划。

### 默认倾向强制反转

我的写作默认倾向和番茄爆款要求是**反的**。每次写新章节时，默认倾向会自动生效。必须强制反转：

| 维度 | 我的默认倾向（❌） | 番茄爆款要求（✓） | 反转方法 |
|------|------------------|------------------|---------|
| 冲突类型 | 对话/探测/敲门 | 物理对抗/战斗 | 每章至少一个"手碰到金属"的动作 |
| 反派行为 | 温和/做梦/被动 | 主动出击/造成伤害 | 反派在前3章内必须已出手 |
| 爽点风格 | 文青爽（理解/和解/开门） | 直给爽（反杀/碾压/打脸） | 每章爽点必须能用动词描述 |
| 结尾方式 | 安静/抒情/画面定格 | 危机/反转/悬念爆炸 | 每章最后一段必须让读者心跳加速 |
| 情绪曲线 | 全程温和/缓慢升温 | 紧张→代价→新威胁 | 日常最多连续2章 |
| 主角行为 | 观察/分析/对话 | 主动出击/物理介入 | 每章主角至少一个"动手不动口"的瞬间 |

### 141-150第一版vs重写版对照（存档为警示）

| 维度 | 第一版（被否决） | 重写版（通过） |
|------|----------------|-------------|
| 10章战斗次数 | 0 | 4（拔电源/短路焊封/缓存短路/冲压撞门） |
| 反派主动出手 | 0次（全程在门里翻身/做梦/回答） | 13次写入攻击+12端口围城+身体改写 |
| 爽点类型 | 文青爽（原稿碎片/门外有人/豆浆） | 直给爽（第一滴血/13页全丢/剁手/破门） |
| 生死局 | 0次 | 2次（身体设定被改写+12端口同时围城） |
| 结尾钩子达标率 | 20%（只有2章炸） | 90%（9章危机/悬念/战斗结尾） |
| 作者形象 | 孤独囚徒，等豆浆 | 主动攻击的改写机器，被打退13次后才降 |

**这组对照必须记住。下次写新章节时，先对照第一版的错误清单——确认自己不是在重蹈覆辙。**

### 141-150重写中发现的6条新规则

**规则1：每10章必须有≥4次物理对抗**

物理对抗定义：主角或核心角色用自己的手/工具触碰对方的物理介质（拔电源/短路/焊封/撞门/枪杆压铜口/冲压机当撞锤）。纯数据对抗（探测/代码/频率）不算。纯对话不算。

**规则2：反派必须在第1-2章内就造成可见伤害**

不能让反派"在梦里翻身""半梦半醒""在门那边打字"。读者需要在前两章就看到反派做了什么具体坏事——擦除了谁、改写了什么、破坏了哪座城。反派不造成伤害=全书无紧张感。

**规则3：爽点必须能用动词描述**

验证标准：这个爽点能不能用"主角/团队做了什么具体动作"来描述？
- ✓ "主角拔了服务器电源，配角的名字回来了"
- ✓ "矛手用焊疤环压在铜口上，焊接师傅焊死了端口"
- ✗ "作者终于说出了'门外有人'"（没有动作，只有对话）

**规则4：章节结尾最后一句话必须制造紧迫感**

结尾不是休止符，是推进器。写完每章后检查：最后一段如果让读者心跳平稳→不合格。合格的结尾会让读者产生"下一页！"的冲动。
- ✓ "第一滴血。但打火机的火焰还没收。"
- ✓ "它开始碰AI了。不是先删人——先删不会写字的系统。"
- ✗ "明天继续。"（心跳平稳→划走）

**规则5：每10章必须有1次"读者真的觉得主角要输"的瞬间**

不能是"差点输但读者知道不会"。必须是读者自己也开始怀疑"这怎么赢？"
- 144章：老焊的焊把不会握了，小欧的手指不认识面团，矛手肌腱被退回撕裂——读者会想"如果身体都能被改写，还有什么能挡住？"

**规则6：信息密度用"动作/对话/描写"三轮换控制**

每300字检查一次：如果连续300字全是设定讲解/内心独白/抽象概念→立即插入一个物理动作或一句对话。
- ✗ 连续5行解释"夹层/废稿/第八本/缝隙/端口"的机制
- ✓ 拆成：1句设定+1个动作（焊条刻字）+1句对话（"它怕物理介质"）

## 阶段十一：三次重写根因分析——为什么规则总是被突破

### 核心发现：默认倾向不是"偏弱"，而是"完全相反"

经过了三次完整重写（131-140、141-150、151-160），每次都写了完整规则、每次第一版都100%重复相同错误。根因已经明确：

**我写作的默认模式和番茄爆款要求是两极对立的关系。**

```
我的默认模式：静谧→对话→发现→理解→和解→抒情收束
番茄爆款要求：冲突→战斗→反杀→碾压→生死局→炸弹收束
```

这不是"需要调整"级别的偏差——这是180度反向。每次写新章节时，默认模式会自动覆盖所有规则，让我写出一版安静的、文学化的、没有战斗的章节。然后用户指出，我重写，加上战斗。第三次了——模式没有改变。

### 元规则：从"写完检查"改为"写前阻断"

所有之前的规则（阶段八、九、十）都是"写完检查"。但写完检查有个致命缺陷：当我已经写了2000+ CJK的安静章节后，不可能通过"加一点战斗"来修复——需要推翻重写。

**唯一有效的策略：写每一章之前阻断默认倾向。**

### 写前单章阻断五问（每章动笔前必须过）

和阶段十的10章五问不同——这次是**每章**都要问。一章一问。

```
问1：这一章反派造成了什么可见伤害？（不是威胁，不是探测，不是对话——是伤害）
      必须具体到：谁受伤了、什么东西被破坏了、哪个地方被改写了。
      答不出→不准写。

问2：这一章的物理对抗是什么？（手碰到金属/火焰/焊条/枪杆/拳头——纯对话不算）
      对抗必须改变物理局面——不是"他说话了"而是"他把东西砸了/烧了/焊死了/打断了"。
      答不出→不准写。

问3：这一章的爽点能不能用一个动词短语描述？（"反杀了XX""碾压了XX""打脸了XX"）
      不能用"发现了XX""理解了XX""看到了XX""对话了XX"——这些不是爽点。
      答不出→不准写。

问4：这一章结尾最后一句话会不会让读者心跳加速？
      检查方法：遮住前面内容，只看最后一段。如果读完感觉"安静""温暖""感动"→不合格。
      合格的感觉是"他要干什么？""下一页！""完了！"。
      不合格→不准写。

问5：这一章前300字有没有物理事件？（不是描写，不是回忆，不是对话——是正在发生的动作）
      如果前300字是"XX站在XX看着XX想着XX"→读者3秒划走。
      前300字必须有：炸了/断了/打了/冲了/倒了/烧了。
      没有→不准写。
```

### 151-160三次重写对照（存档为永久警示）

| 维度 | 第一版（一次） | 第二版（二次） | 第三版（通过） |
|------|-------------|-------------|-------------|
| 开场 | 蓝柜开，铜箔网络全通 | 蓝柜开，全通→敌敲门 | 蓝柜开→白光打穿码头，老拐断臂 |
| 反派首次出手 | 第152章敲门 | 第153章探测 | 第151章即开战，物理改写攻击 |
| 第一次战斗 | 无 | 第153章火烧光丝 | 第151章焊机电磁噪波反击 |
| 爽点类型 | 开柜/联网/对话 | 开柜/敲墙/和解 | 碳灰反杀+合金骗敌自伤=2次反杀 |
| 伤亡 | 无 | 无 | 老拐骨折+货柜砸毁+铁环崩 |
| 结尾钩子达标 | 20% | 40% | 80% |
| 主线目标 | 开柜找原稿 | 开柜对战→谈判 | 被攻击→反杀→发现敌人是狱友 |
| 重写原因 | 文青爽/无战斗 | 文青爽/温和对抗 | — |

**这组对照证明：只要前300字没炸，这章就废了。第一版的蓝柜开和全通网络都是"好事"——番茄读者不喜欢"好事"，他们喜欢"炸事"。**

### 第三版成功的本质：把"发现"改成"被攻击"

核心改动只有一条——把每章的开头从"他们发现了XX"改成"敌人攻击了XX"。这是一个可复制的公式：

```
❌ 开章公式（文青爽）：角色X发现了Y，这个Y意味着Z，他们决定……
✓ 开章公式（番茄爆款）：敌人用X攻击了Y，造成了Z的伤害，角色反击……
```

151章的变化就是这句话：
- ❌ "蓝柜打开了，里面是铜箔网络图"（发现）
- ✓ "蓝柜打开后零点三秒，白光打穿了码头，老拐左臂断了"（被攻击）

### 反派伤害可见性检查

从此以后，反派每一次出手必须造成一个"可见伤害"——读者能在脑子里看到画面的伤害。

```
不可见伤害（❌）：数据被改写、频率被干扰、字段归零、温度异常
可见伤害（✓）：骨头断了、货柜砸了、钢缆崩了、墙面熔了一个洞、铁砣飘起来

规则：每次反派出手，至少产生一个可见伤害。可见=能用手机拍照拍到的。
```

### 爽点动词化检查

从此以后，"爽点"必须是动词短语，不能是名词或状态。

```
❌ 不是爽点：开门、联网、原稿碎片、作者对话、和解、理解、编号交换
✓ 是爽点：反杀了对方的端口、骗敌自伤、用碳灰污染了笔尖、打断了他的攻击

规则：每章的爽点必须能描述为"谁对谁做了什么，导致了对方的损失"。
损失=端口崩了/攻击灭了/武器废了/人被击退了。
```

## 阶段十二：智斗依赖症——第四次重写根因（161-170）

### 核心发现：我的默认爽点模式是"智斗"，不是"爆发"

经过了131-140（文青爽）、141-150（文青爽）、151-160（文青爽→战斗版）、161-170（智斗克制版）四次完整迭代。第四次终于不再文青，但又掉进了另一个坑：**智斗依赖**。

```
第一次（131-140）：纯文青——和解/豆浆/焊光/明天继续
第二次（141-150）：探测/敲门/原稿/对话——文青变体
第三次（151-160）：战斗版——但战斗是"骗敌自伤""合金卡死""借笔杀端口"
第四次（161-170）：智斗版——全程陷阱/反刺/欺骗/碳标，0次正面硬刚
```

**四次重写的共同根因：我的爽点是"聪明"而不是"暴力"。**

番茄读者要的不是"主角用了什么巧妙的计谋"——是"主角碾碎了敌人"。他们不要在脑中解码三层陷阱逻辑，他们要看到敌人被正面轰飞。

### 智斗和爆发的本质差异

| 维度 | 智斗（我的默认，❌） | 爆发（番茄要的，✓） |
|------|-------------------|-------------------|
| 爽点来源 | "我骗了你" | "我碾了你" |
| 读者反应 | "哦，好聪明" | "草！打得好！" |
| 情绪 | 克制、冷静 | 释放、痛快 |
| 战斗方式 | 陷阱/欺骗/反刺/碳标 | 正面刚/硬碰硬/一拳打回去 |
| 反派感受 | 困惑/被误导/被算计 | 恐惧/痛苦/被碾压 |
| 阅读体验 | 需要想一下才爽 | 3秒内爽到 |

**161-170的罪：全程"骗"——假伤骗、碳灰骗、甲片骗、暖波骗。没有一次正面把掠食者打飞出去。读者积累的愤怒没有被释放——他们想看到掠食者被打爆，不是被聪明地骗走。**

### 爆点前移铁律

1-5章必须至少有一次正面爆发。不能把爆发全部堆在7-9章。

```
161-170原版节奏：铺垫(1-5) → 试探(6) → 爆发(7-9) → 收束(10)
番茄要求节奏：爆发(1-2) → 升级(3-4) → 大爆发(5-6) → 反杀(7-8) → 新威胁(9-10)
```

**161-170错的不是内容，是顺序。同样的战斗——如果把167章掠食者首现+168章甲片反刺移到第162-163章，把164-165的旧伤分析和造饵压缩到一章，然后169-170放更大的爆发——节奏立刻不一样。**

### 最大缺失：没有碾压和装逼

回顾全部四批章节，**碾压场景=0，装逼场景=0**。这是番茄最刚需的两种爽点，一次都没写过。

碾压定义：主角展示压倒性实力，敌人毫无还手之力。
装逼定义：主角轻描淡写做了一件别人做不到的事，周围人震惊。

```
碾压范例（应写未写）：
- 矛手一枪捅穿掠食者的手脸，把它钉在岩壳上→它挣扎，枪杆纹丝不动
- 主角把打火机打着，掠食者看到火焰往后退了三步——它怕火
- 焊接师傅用焊机一弧切断掠食者一根指甲——"比焊条好切"
```

### 每章结尾"炸"的硬标准

从此以后，章节结尾分三档。只有第一档合格。

```
一档（✓）：敌人突然出现/攻击/变故——读者心跳加速，立刻翻页
二档（△）：决定/计划/倒计时——读者知道接下来有动作，但不紧迫
三档（✗）：安静/交代/画面定格/感悟——读者放下手机

161-170达标率：一档6章，二档2章，三档2章
番茄要求：一档≥8章/10章
```

二档结尾的问题："明天继续""倒计时开始""碳标永续"——这些都告诉读者"好戏在后面"，但番茄读者要的是"好戏在下一页"。他们不会等明天。

### 智斗的正确用法：爆发前菜，不是主菜

智斗可以写，但必须满足两个条件：
1. 智斗之后立即接正面爆发（让读者看到计谋的瞬间结果）
2. 智斗不能超过一章——一章想、一章做、一章炸

```
正确节奏：
164章：分析旧伤，发现弱点（智斗前菜）
165章：正面迎击，第一次打退掠食者（爆发主菜）——不是骗退，是打退
❌ 161-170实际：164-166全在造饵布阱，168才爆发——前菜吃了三章
```

### 从下一批开始追加的铁律

1. **1-2章内必须有正面爆发**——不能让读者等超过两章才看到第一次打
2. **每10章至少1次碾压或装逼**——主角或核心角色展示压倒性实力
3. **智斗≤1章**——分析/造饵/布阱/欺骗全部压缩到一章内
4. **结尾一档率≥80%**——每10章至少8章用敌人/攻击/变故收尾
5. **爆发必须是正面硬刚**——不能全部靠陷阱和欺骗，至少2次直接对抗

## 阶段十三：全卷审稿实战教训——49章大修（起点平台）

### 核心元教训：起点 ≠ 番茄

用户明确纠正：**"这是起点爆款，不要按照番茄来。"** 两者的核心差异：

| 维度 | 番茄爆款 | 起点爆款 |
|------|---------|---------|
| 读者耐心 | 3秒划走 | 愿意读慢热设定 |
| 爽点密度 | 3章一爽 | 可以10章逐步铺垫 |
| 文风 | 短句/直给/动词轰炸 | 可以有一定文学质感 |
| 主角 | 全程碾压 | 允许成长弧线、恐惧、犯错 |
| 群像 | 主角视角为主 | 群像可以充分展开 |
| 历史/设定 | 简化 | 可以厚重、有考据感 |
| 结尾钩子 | 每章必须炸 | 悬念可层层递进 |
| 情感模式 | 撕裂式爆发为主 | 安静的感动也可以成立 |

**但两者共同底线**：不能水、不能模板化、不能AI味、每章必须有钩子。

### 教训1：开头动词检查的实战盲区

检查规则是"前80 CJK必须有动作动词"，但实际执行中容易出问题：

**盲区1**：环境描写占了开头——如"黑雾退去之后粮车的废墟还在闷烧。残烟从焦壳缝隙里一缕一缕往上飘……"——景物很美，但没有角色动作。读者等了好久才看到"主角坐在……"

**修复公式**：`角色+动作+环境`——先把人扔进场景再做描写。
```
❌ "黑雾退去之后粮车的废墟还在闷烧。残烟…主角坐在城墙垛口边的石墩上。"
✓ "主角把矛从水渠边的土里拔出来，矛尖上还挂着半干的黑雾残胶。他把矛杆横搁在膝上，在城墙垛口边的石墩上坐下来。…粮车的废墟还在闷烧。"
```

**自动化检查的局限**：perl脚本只能检测"前80 CJK是否包含动词字符"，但无法判断这个动词是角色主动发出的还是环境被动描述的。环境描写中的"飘""蔓延""流淌"等也会被识别为动词。**人工确认不可替代。**

### 教训2："不是X是Y"要区分AI模板和口语自然用法

之前把"不是X是Y"一刀切为AI味模板，实际执行中发现：

**是AI模板的信号**（要改）：
- "不是恐惧，而是某种更深层的东西"——抽象、空洞
- "不是因为A，而是因为B"——解释性太强

**是口语自然用法**（可以保留）：
- "不是麻绳，是已经褪成灰白色的细丝绳"——具体物品辨识
- "不是矛杆。矛杆是郑矛用残魂借给他…这把剑是他的。"——角色内心判断
- "不是画面，是声音"——感知辨别
- "不是被杀的"——事实陈述

**判断标准**：如果"不是X是Y"中X和Y都是具体可感知的名词/动作→保留；如果X和Y都是抽象概念/情绪→修改。

### 教训3："他感到"类叙事中介的三种改造法

全书从最初大量"他感到"清理至3处残留，最终全部清零。改造公式：

**改造A：直接呈现**（最常用）
```
❌ "他感到臂后侧肌肉群被一股不属于他的力量往后拉"
✓ "臂后侧肌肉群被一股不属于他的力量往后拽——他自己的神经还在，但控制权被那道残魂接管了"
```

**改造B：动词替代**（"感到"换成具体动作动词）
```
❌ "然后他感到手掌下有一股反向力"
✓ "然后手掌下撞来一股反向力"
```

**改造C：因果链接入**（把"感到"拆成物理因果链）
```
❌ "他感到符石底下的渗透波在被封死前发出了一段连续的紊乱脉冲"
✓ "符石底下的渗透波在被封死前发出了一段连续的紊乱脉冲，从石面直接撞进他的掌骨"
```

**注意**：改造后必须检查CJK——直接呈现/因果链通常比"他感到"字数更多，一般不会缩水。

### 教训4：修改后必须四维全检

每次修改（即使是几个字的小改）都可能影响：
1. **CJK字数**——替换后字数可能增减
2. **破折号数量**——新增的因果链可能引入新破折号
3. **开头动词**——调整段落顺序会改变开头80字
4. **结尾钩子**——修改正文可能意外改变最后一段

**本次发现的问题都是在修改后重新四维全检时发现的**（第030章开头动词在之前的修改中被环境描写覆盖掉了）。

**不可跳过的检查顺序**：CJK → Dash → 开头动词 → 结尾钩子 → 新增内容的逻辑一致性。

### 教训5：跨章节一致性——加料后的"多米诺检查"

在多个章节插入新内容后（本次加了十处"名场面"），必须检查新内容和原有内容的交叉影响：

**检查清单**：
- [ ] 角色状态连续性：A章角色受伤→B章他的行动描述是否反映了这个伤？（老井第040章肺伤→第049章挑土，需补"呼吸很稳，肺里不再有杂音了"）
- [ ] 重复描写：同一动作/场景被描写两次？（第006章水银人形"解散自己"后又写"收手溶回"）
- [ ] 物品携带：角色在A章拿到的物品，B章是否还在身上？
- [ ] 时间线：加料中的时间锚点（"第二天""三天后"）是否和前后章一致？

**方法**：加料后画出"影响链"——每个加料点往前查到上一处状态描写，往后查到下一处相关描写，确认中间没有断裂。

### 教训6：收束章模板化是最隐蔽的病

收束章（每批末尾、偶数位置、战后/过渡章节）最容易掉进模板。**跨项目共性病**：X0章群像模板（角色A归档→矛手放矛→女孩画画→保安调频→清洁工合手册→主角转物件）和偶数章战后模板（休整→状态检查→听汇报→防线态势→新发现→"在"字结尾）。

**诊断方法**：检查所有偶数章/收束章的最后一段动词。
- 如果连续3章以上结尾动词相同（如本次发现的8章"在"字结尾）→模板已形成
- 读者可能不会逐字分析，但潜意识会发现规律→跳过

**修复**：每章收束聚焦不同的1-2个角色，用他们的独有动作收尾。不要每章都让主角"在"某个地方观察某个东西。

### 教训7：角色的"代价"不需要每次都死人

起点读者不要求每80章死一个人。但他们需要感受到"代价的重量"。

**本次用三种方式兑现了代价**：
1. **重伤未死但留下痕迹**：老井被源质入肺（第040章），恢复后呼吸有变化（第049章"肺里不再有杂音了"——留痕）
2. **历史中的牺牲者**：旧秦军守粮兵骸骨+木牍"粮仓守三日。水从暗沟退了。勿念。"（第049章）——不是主角团的牺牲，但情感重量不轻
3. **主角的身体代价**：主角掌心毛细血管被源质挤破（第049章"血不多，只在掌纹里浅浅地漾了一层"）

**原则**：代价必须"可见"。血、伤疤、骸骨、木牍上的字——读者能在脑子里看到画面。

### 教训8：python3路径问题的实战教训

Windows环境下`python3`命令可能是App Store重定向器而非真实Python。检测方法：
```bash
# 先用 which/where 确认路径
which python3
# 如果指向 WindowsApps → 用完整路径
/c/Users/Lay/AppData/Local/Programs/Python/Python312/python.exe
```

### 从下一本开始追加的铁律

1. **平台确认**：动笔前确认是起点还是番茄——两者爽点密度、文风、群像展开方式不同
2. **修改后四维全检**：每次修改（哪怕改几个字）→立即跑CJK+Dash+动词+钩子
3. **加料后一致性检查**：每个新加段落→查前查后→确认状态连续
4. **收束章动词轮换**：偶数/收束章结尾动词不能连续3章相同
5. **"不是X是Y"不再一刀切**：具体可感知→保留；抽象概念→改
6. **代价可见化**：每次代价描写→读者能否在脑中拍照？
7. **开头段落角色优先**：第一句必须是角色在对物理世界做动作——环境可以跟在后面

## 阶段十四：配角饱满度铁律——55章实战教训

### 核心元教训：主角不能一个人扛所有高光

实战项目55章后复盘发现：主角的高光密度远超配角（每2-3章一次战斗/情感高光），而十人队伍中有5人的饱满度严重不足。半数配角几乎没有独立存在感。

根因：写作时默认把"高光"分配给主角。配角只在需要他们推进情节时出现——这是工具人化。群像文读者的核心心理是"这群人在一起才无敌"——如果只有主角一个亮眼，队伍就变成了跟班。

### 配角饱满度三层模型

每个重要配角必须有三样东西：

| 层次 | 定义 | 最少频次 | 示例 |
|------|------|---------|------|
| **独立高光** | 这个角色做了一件只有他能做、换别人做不了的事 | 每20章至少1次 | 机尉把手伸进暗沟听防溃线（第051章） |
| **低谷/弱点** | 这个角色不体面的瞬间——怕、犯错、崩溃、被打垮 | 每人至少1个已揭露的弱点 | 田必"我怕矛"、老井被源质吸到濒死 |
| **独立故事碎片** | 不完整的过去片段——不是在对话里交代，是通过物件/动作/回忆闪回 | 每卷每人至少1个碎片 | 郑矛矛杆上的女人发绳蝴蝶结、小孩攥着碎石片学刻防溃线 |

### 每10章配角分配铁律

在10章大纲阶段，必须检查以下分配表：

```
第XX-XX章配角配置：
| 角色 | 独立高光章 | 低谷展示章 | 独立物件/动作 |
|------|-----------|-----------|-------------|
| 角色A | 第X章（具体做了什么） | 第Y章 | 某物/某习惯 |
| 角色B | … | … | … |

强制要求：
- [ ] 10章中至少3个不同配角有独立高光（不能全是主角）
- [ ] 主角的高光章≤5章/10章（留出空间给配角）
- [ ] 至少2个配角展示低谷（不是说出来的弱点，是画面上能看到的）
- [ ] 上次10章中未获高光的配角，本次优先分配
```

### 配角高光≠战斗力展示

配角的高光不一定是战斗。可以是：

- **技能高光**：配角A用回声给每块砖建档——别人做不到
- **情感高光**：配角B把"配角B韩地"纸条压在无名骨架旁边——十五岁小孩独有的告别方式
- **物件高光**：机尉把弩机里的石片抽出来垫高碎骨毡布——"和别的骨架一样高了"
- **沉默高光**：田必把矛放门外空手进死人坑——怕矛的人的静默选择
- **身体语言高光**：赵仲没用弓臂接小孩的骨头——弯腰伸手直接抱

### 主角的低谷不能缺席

主角不仅需要高光——还需要低谷。全55章回顾：主角的低谷只有第028章第一次杀人哭了和第022章砸龙柱。其余时间他都是"正确的决策者"。

**每15-20章必须给主角安排一次低谷**——他搞砸了什么、判断错误导致损失、被队友救了、恐惧到动弹不得。低谷中让配角拉扯他一把，这是最自然的角色关系建立方式。

### 配角独立故事的碎片插入法

不另立章节，在战斗/过渡章中插入1-2段碎片：

```
格式：角色正在做动作 → 触发一个物件/气味/声音 → 1-2句过去 → 切回现在
示例（055章）：田必右手按石壁水位线 → "水退了"——和木牍上守粮兵写的差三个字
```
